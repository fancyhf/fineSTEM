### 方案一：基于现有AI IDE的“教学型项目模版” (最轻量，推荐起步)
利用 Cursor / Trae 本身强大的对话能力，通过**配置文件**和**项目结构**，把IDE变成老师。

*   **核心逻辑：**
    你分发给学生的不是一个普通的代码包，而是一个**“配置好人设”的项目包**。
*   **如何实现“嵌入”：**
    1.  **预设 System Prompt (如 `.cursorrules` / `.trae/rules`)：**
        *   在项目根目录下放一个配置文件，写死AI的人设：“你不是代码生成器，你是物理老师。当学生问代码问题时，不要直接给答案，要结合抛物线公式（y = ax² + bx + c）来解释代码逻辑。”
    2.  **埋点式注释 (Interactive Comments)：**
        *   在关键代码行（比如重力参数 `g = 9.8`）旁边写注释：`// TODO: 试着把这里改成 1.6 (月球重力)，然后问AI会发生什么？`
*   **用户体验：**
    学生打开项目 -> 看到注释引导 -> 在IDE右侧Chat输入“我改了重力，为什么球飞得更高了？” -> AI根据预设人设，解释重力对加速度的影响。

### 方案二：Web端“交互式游乐场” (体验最好，适合自媒体展示)
开发一个简单的Web App，左边是交互界面，右边是**“限制版”的AI Chat**。

*   **核心逻辑：**
    前端界面 + 调用大模型API (DeepSeek/OpenAI)。
*   **如何实现“嵌入”：**
    *   **界面：** 左边是物理实验模拟器（比如一个小球在跳），右边是一个聊天框。
    *   **上下文注入：** 当学生在聊天框提问时，系统自动把**当前的代码片段**和**变量状态**（比如当前速度、高度）打包发给AI。
    *   **场景：** 学生问“球怎么停了？”，AI能回答：“因为检测到 `speed` 变量降到了0，你看第45行的 `if` 判断语句...”
*   **价值：** 这种形态非常适合做成SaaS服务或课程配套软件，不用学生配环境，打开浏览器就能用。

### 方案三：浏览器插件伴侣 (通用性最强)
开发一个Chrome插件，专门配合 Github Codespaces 或 Bolt.new 使用。

*   **核心逻辑：**
    当学生浏览特定编程网页时，插件激活，提供侧边栏助手。
*   **缺点：** 开发成本较高，且依赖第三方平台稳定性，暂不推荐作为第一步。
