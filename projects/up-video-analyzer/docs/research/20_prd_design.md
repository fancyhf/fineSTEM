# 需求与设计文档：UP 主视频内容分析器

> 项目: up-video-analyzer  
> 创建时间: 2026-02-28  
> 文档版本: v1.0.0

---

## 1. 需求范围

### 1.1 Must-Have (必须有)

| # | 功能 | 说明 |
|---|------|------|
| 1 | 字幕文件上传 | 支持 SRT/TXT 格式 |
| 2 | 中文分词 | 使用 jieba 进行中文分词 |
| 3 | 词云生成 | 生成可视化词云图 |
| 4 | 统计面板 | 展示词频统计、总词数等指标 |
| 5 | 导出报告 | 支持 PNG 格式导出 |

### 1.2 Nice-to-Have (锦上添花)

| # | 功能 | 说明 |
|---|------|------|
| 1 | AI 智能总结 | 调用大模型 API 生成内容总结 |
| 2 | 多视频对比 | 支持多个视频的分析对比 |
| 3 | 情感分析 | 分析字幕情感倾向 |
| 4 | 时间轴分布 | 关键词在时间轴上的分布 |

### 1.3 Won't-Do (明确不做)

| # | 功能 | 原因 |
|---|------|------|
| 1 | 视频自动下载字幕 | 技术复杂，时间不足 |
| 2 | 实时视频分析 | 超出时间预算 |
| 3 | 用户账户系统 | 非核心需求 |
| 4 | 云端存储 | 本地应用即可满足需求 |

## 2. 验收标准

| # | Given | When | Then |
|---|-------|------|------|
| 1 | 用户上传 SRT 字幕文件 | 点击分析按钮 | 显示词云图、统计面板 |
| 2 | 用户上传 TXT 文本文件 | 自动触发分析 | 生成完整报告 |
| 3 | 用户点击导出 | 选择 PNG 格式 | 下载词云图片 |
| 4 | 用户输入视频链接 | 点击确认链接 | 检测平台并显示选项 |
| 5 | 用户查看任务历史 | 点击任务历史标签 | 显示历史任务列表 |

## 3. 技术设计

### 3.1 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Streamlit | Python原生，快速开发 |
| 分词库 | jieba | 中文分词标准库 |
| 词云库 | wordcloud | 功能强大，易于定制 |
| 数据处理 | pandas | 数据处理标准库 |
| 图表 | matplotlib | 与wordcloud配合良好 |

### 3.2 替代方案对比

| 方案 | 优点 | 缺点 | 选择 |
|------|------|------|------|
| Streamlit | 简单快速 | 功能有限 | ✅ 选用 |
| Flask + Vue | 功能强大 | 学习成本高 | ❌ 放弃 |
| Gradio | 适合ML Demo | 布局不够灵活 | ❌ 放弃 |

### 3.3 系统架构

```
用户输入
├── 字幕文件上传 (SRT/TXT)
└── 视频链接输入 (URL)
    └── 平台检测
        ├── B站 → CC字幕提取 / 语音识别
        ├── 抖音 → 语音识别
        └── YouTube → 语音识别

数据处理
├── 字幕解析 → 纯文本
├── 中文分词 (jieba)
├── 停用词过滤
└── 词频统计

输出展示
├── 词云图 (wordcloud)
├── 统计面板 (pandas)
└── 导出功能 (PNG)
```

---

## 4. 数据流设计

```
Input: 用户上传字幕文件
  ↓
Parse: 解析 SRT/TXT 格式，提取纯文本
  ↓
Tokenize: jieba 中文分词
  ↓
Filter: 停用词过滤 (1000+ 停用词)
  ↓
Count: 词频统计 (Counter)
  ↓
Visualize: 生成词云图 (wordcloud)
  ↓
Display: Streamlit 展示结果
  ↓
Export: 导出 PNG 图片
```
