# 开题报告（10_proposal）

> 用途：把“想法”变成可执行的研学课题。当前版本基于已完成的脑爆与立项信息整理。

- 项目名称：文学知识卡 - 二次元漫画版
- 研学主题/领域：AI 辅助语文学习 / Web 教育应用
- 学生/小组：未来科技学院单人项目（初中组）
- 年龄段：初中
- 预计周期：12h+
- 版本：v1
- 日期：2026-03-03

## 1. 背景与动机（≤150字）
- 古诗词学习常见痛点是“背得快、忘得也快”，纸质笔记检索困难，复习效率低。本项目希望用卡片化结构整理诗词知识，再用二次元插画提升记忆兴趣，让同学在“看、搜、收藏、再复习”的闭环中更主动地学习语文。

## 2. 研究问题（1–3条）
- RQ1：卡片式学习是否能降低古诗词复习时的检索成本？
- RQ2：二次元视觉表达是否能提升学生对诗词内容的记忆意愿？
- RQ3（可选）：网页导入与自动搜集功能是否能持续扩展知识库质量？

## 3. 目标与成功标准（至少2条，可测）
- 标准1：核心功能可用；通过条件：完成“录入 + 搜索 + 分类 + 卡片展示 + 收藏”闭环并可在本地访问。
- 标准2：界面风格统一；通过条件：整体页面保持卡哇伊配色与卡片式布局，关键页面视觉一致。
- 标准3：可演示；通过条件：能够现场演示至少 5 个操作场景（首页浏览、搜索、筛选、收藏、导入）。

## 4. 方法概述（≤200字）
- 一句话流程：用户输入/导入诗词 → Flask 处理与存储 → 卡片化展示与检索 → 可选生成二次元插画辅助记忆。
- 关键做法（2–4点）：
  1) 使用 Flask + Jinja2 快速搭建本地 Web 学习工具，降低开发门槛。
  2) 以 JSON 文件保存诗词数据，便于快速迭代与调试。
  3) 将“搜索、分类、收藏、导入”作为学习闭环核心操作。
  4) 将 AI 插画设为增强功能，保障在 API 不可用时仍可完成主线学习体验。

## 5. 资源与约束
- 可用资源（设备/数据/工具）：Python、Flask、Jinja2、requests、BeautifulSoup、本地 JSON 诗词库、AI IDE。
- 关键约束：总时长 12h+；优先本地运行；外部 AI API 可用性不稳定；不引入数据库与复杂账号系统。
- 不做清单（wont-do，≥2）：用户注册登录系统；多人协作功能。

## 6. 风险与应对（至少2条）
- 风险1：AI 漫画生成功能受 API 稳定性和配额影响。→ 应对/替代方案：默认支持无插画学习流程，失败时回退为占位图或文本卡片。
- 风险2：功能逐步增多导致调试成本上升。→ 应对/替代方案：按里程碑推进，每完成一步就验证搜索/展示主流程。

## 7. 里程碑计划（2–4个）
- M1：基础框架可运行（预计：0.5h）
- M2：核心学习功能可用（预计：3h）
- M3：高级功能与 AI 插画增强（预计：5h）
- M4：整体验收与展示准备（预计：1.5h）

## 8. 伦理与安全（必填）
- 数据与隐私：仅处理公开诗词与学习内容，不采集敏感个人信息。
- 硬件安全（如适用）：本项目为纯软件 Web 项目，不涉及硬件操作风险。
- 学术诚信：所有引用诗词与素材来源需标注，AI 生成内容需明确“AI辅助”属性。
