# 原型设计说明书（30_prototype_spec）

> 用途：说明“界面/交互/流程”长什么样，便于快速验收与迭代。

- 项目：文学知识卡 - 二次元漫画版
- 版本：v1
- 日期：2026-03-03

## 1. 原型目标（≤80字）
- 验证诗词“浏览—搜索—筛选—收藏—导入”的学习流程是否顺畅，并评估卡哇伊卡片风格是否利于记忆。

## 2. 信息架构（IA）
- 信息分区：学习浏览区、内容管理区、学习增强区
- 导航方式：顶部导航 + 页面跳转链接

## 3. 页面/界面清单（建议 ≤3–4）
### 页面1：首页（诗词卡片学习）
- 目的：
- 以卡片方式集中展示诗词，支持搜索、分类与收藏。
- 入口：
- 用户访问根路径 `/`。
- 组件清单：
- Header、SearchBar、CategoryFilter、PoetryCard、FavoriteButton。
- 交互说明（点击/输入/反馈）：
- 输入关键词后触发搜索；选择分类后局部刷新结果；点击收藏后状态即时变化。
- 边界情况（至少1条）：
- 搜索无结果时显示空状态提示，不展示空白页。

### 页面2：新增页面（手动录入）
- 目的：
- 让学生快速录入新的诗词或文学知识卡片。
- 入口：
- 首页“新增”按钮，路由 `/add`。
- 组件清单：
- AddForm（标题、作者、朝代、正文、分类）、提交按钮。
- 交互说明（点击/输入/反馈）：
- 提交后进行最小校验，成功则跳回首页并显示新增内容。
- 边界情况（至少1条）：
- 必填项为空时阻止提交并给出错误提示。

### 页面3：导入与增强（可选高级）
- 目的：
- 通过网页导入和 AI 插画生成提升知识库扩展效率与学习兴趣。
- 入口：
- 路由 `/import` 与卡片上的“生成插画”操作。
- 组件清单：
- ImportForm、导入结果提示、GenerateImageAction。
- 交互说明（点击/输入/反馈）：
- 输入 URL 后触发导入接口，成功回显导入结果；生成插画后更新卡片 image_url。
- 边界情况（至少1条）：
- 链接无效或提取失败时显示具体失败信息并保持页面可继续操作。

## 4. 关键用户流程（文字流程图）
- 流程A：打开首页 → 浏览卡片 → 输入关键词搜索 → 选择分类筛选 → 收藏重点诗词 → 完成复习清单。
- 流程B：进入新增页/导入页 → 录入或导入诗词 → 返回首页确认展示 → 进入统计页查看总量与分类分布。
- 流程C：点击“生成插画” → 调用 AI 图像接口 → 成功返回 imageUrl 并更新卡片，失败则展示错误提示并保留主流程可用。

### 4.1 流程说明（系统序列）
用户操作（浏览/搜索/新增/导入/收藏/生成插画）  
→ Flask 路由接收请求  
→ 业务处理（JSON 读写或外部接口调用）  
→ 返回页面或 JSON 响应  
→ 前端刷新并反馈状态

## 5. 文案与反馈（UX）
- 成功提示：
- “成功添加诗词”；“成功导入诗词《xxx》”；“二次元插画生成成功”。
- 错误提示：
- “请输入网页链接”；“导入失败：xxx”；“生成失败：请检查 API 配置或网络连接”。
- 空状态：
- “暂无匹配诗词，请尝试更换关键词或分类”；“等待导入第一条内容”。

## 6. 验收用例（≥3）
- 用例1：Given 用户访问首页，When 页面加载完成，Then 显示卡片列表和搜索控件。
- 用例2：Given 用户输入“李白”，When 点击搜索，Then 返回作者或内容匹配“李白”的卡片。
- 用例3：Given 用户选择“唐诗”分类，When 执行筛选，Then 仅显示分类为“唐诗”的内容。
- 用例4：Given 用户点击收藏按钮，When 刷新后进入收藏页，Then 仍可看到已收藏卡片。
