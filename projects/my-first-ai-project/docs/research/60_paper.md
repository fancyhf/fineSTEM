## Title
文学知识卡 - 二次元漫画版：面向初中生的卡片化古诗词学习 Web 应用设计与实现

## Abstract（≤150字）
- 本研究面向古诗词复习“检索慢、记忆弱”问题，设计并实现了一个卡片化学习 Web 应用。系统采用 Flask + JSON 架构，支持搜索、分类、收藏、导入与自动扩展，并引入二次元插画增强学习兴趣。结果表明，系统已具备可演示的学习闭环能力，但在外部 API 稳定性与安全配置方面仍需优化。

## 1. Introduction
- 背景与动机：传统纸质笔记在古诗词复习中检索效率低、互动性弱，学生持续使用意愿不足。
- 研究问题（RQ1/RQ2）：
  - RQ1：卡片化检索与分类是否能提升诗词复习效率？
  - RQ2：二次元视觉增强是否能提升学习参与度与记忆意愿？
- 贡献点（2–3条）：
  - 构建了面向初中生的轻量学习工具闭环（录入-搜索-收藏-复习）。
  - 将网页导入与自动搜集结合，增强知识库扩展能力。
  - 探索了诗词内容与 AI 插画融合的教学应用场景。

## 2. Method
- 流程概述（配图放 `docs/research/assets/`）：用户输入/导入诗词 → Flask 路由处理 → JSON 存储与检索 → 卡片化展示 → 可选插画生成。
- 关键设计选择与理由：
  - 选择 Flask：实现路径短、学习成本低，适合研学项目快速迭代。
  - 选择 JSON 持久化：数据结构直观，便于学生理解与手工维护。
  - 增强功能解耦：AI 插画失败不阻断主学习流程，提高系统韧性。

## 3. Experiments / Evaluation
- 数据/设置：以内置诗词样本和导入内容作为数据源，本地端口运行 Flask 服务进行交互验证。
- 指标与结果（表/图放 assets）：
  - 功能闭环指标：核心功能可运行并可演示（通过）。
  - 交互可用性指标：搜索、筛选、收藏交互可完成（通过）。
  - 增强能力指标：AI 插画生成受外部服务影响（部分通过）。
- 误差分析（至少1段）：导入功能依赖网页结构，遇到非标准页面时提取准确率下降；AI 图像生成依赖网络与服务稳定性，可能出现超时或响应格式异常，导致增强链路波动。

## 4. Discussion
- 为什么有效/哪里不足：卡片化+检索机制有效缩短了复习路径，但尚未覆盖学习进度追踪、错题复盘与个性化推荐等深层学习支持能力。
- 伦理与安全讨论（必选）：项目主要处理公开诗词文本，隐私风险低；但应避免在代码中保留明文密钥，且 AI 生成图片需标注“AI辅助”并避免版权争议素材。

## 5. Conclusion
- 项目验证了“卡片化诗词学习 + AI 视觉增强”的可行性，并形成可演示原型。后续将重点优化安全配置、导入稳定性与学习评估机制，提升教学落地价值。
