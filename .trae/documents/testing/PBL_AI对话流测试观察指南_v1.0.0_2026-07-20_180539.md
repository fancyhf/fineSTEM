## 1. 环境准备

### 1.1 启动后端服务

```powershell
# 进入后端目录
cd apps/backend

# 安装依赖（如果未安装）
pip install -r requirements.txt

# 启动服务
python -m uvicorn main:app --reload --port 3200
```

**验证后端启动**:
```powershell
curl http://localhost:3200/api/v1/health
```

### 1.2 启动前端服务

```powershell
# 进入前端目录
cd apps/frontend

# 安装依赖（如果未安装）
npm install

# 启动开发服务器
npm run dev
```

**验证前端启动**:
打开浏览器访问 http://localhost:5184

### 1.3 启动 ZeroClaw Gateway

```powershell
# 进入 ZeroClaw 目录
cd H:\dev-env\zeroclaw

# 启动服务
zeroclaw.exe
```

**验证 ZeroClaw**:
```powershell
curl http://127.0.0.1:42617/health
```

## 2. 手动测试观察

### 2.1 测试场景 1: Bootstrap 三问流程

**测试步骤**:

1. **打开 Create 页面**
   - 访问 http://localhost:5184/create
   - 确认页面加载完成

2. **触发第一个问题（年级）**
   - 在聊天框输入: "我想做一个新项目"
   - 按 Enter 发送
   - **观察点**: 
     - 是否出现选项卡片？
     - 卡片标题是否为 "你现在是哪个年级？"
     - 选项是否包含 "初中" 和 "高中"？

3. **选择年级**
   - 点击 "初中" 选项
   - 点击 "下一步" 或 "确定"
   - **观察点**:
     - 发送的消息格式是否为 `[选择:junior] 初中`？
     - AI 是否识别了选择？

4. **触发第二个问题（时间）**
   - **观察点**:
     - 是否出现新的选项卡片？
     - 卡片标题是否为 "你打算花多长时间完成这个项目？"
     - 步骤指示器是否显示 "2 / 3"？

5. **选择时间**
   - 点击 "6小时" 选项
   - 点击确定
   - **观察点**:
     - 消息格式是否正确？

6. **触发第三个问题（想法）**
   - **观察点**:
     - 是否出现 "你有初步想法了吗？" 卡片？
     - 步骤指示器是否显示 "3 / 3"？

7. **选择想法**
   - 点击 "完全没想法，需要脑爆"
   - **观察点**:
     - AI 是否进入脑爆阶段？
     - 回复是否包含脑爆相关内容？

**预期结果**: 三问流程顺利完成，AI 正确识别每个选择并推进到下一阶段。

---

### 2.2 测试场景 2: 回复完整性

**测试步骤**:

1. 在聊天框输入: "请详细解释如何从零开始规划一个完整的 STEM 项目"
2. 按 Enter 发送
3. 等待 AI 回复（约 15-20 秒）

**观察点**:
- 回复长度是否超过 200 字符？
- 回复是否以完整句子结尾？（不应以 "首先"、"然后" 等词结尾）
- 如果有代码块，是否成对出现？（``` 数量为偶数）
- 回复是否被截断？

**预期结果**: 回复完整、连贯，没有被截断。

---

### 2.3 测试场景 3: 多选选项卡

**测试步骤**:

1. 输入: "选择你的兴趣爱好（可多选）"
2. 等待选项卡片

**观察点**:
- 卡片是否显示为多选模式？
- 是否可以同时选择多个选项？
- 再次点击已选选项是否可以取消选择？

3. 选择多个选项后点击确定

**观察点**:
- 发送的消息是否包含所有选中的选项？

---

### 2.4 测试场景 4: AI 识别选择格式

**测试步骤**:

1. 预设聊天历史（通过 API 或手动）:
   - 用户: "我想做一个记账工具"
   - AI: "好的，让我了解一下你的情况。"
   - 用户: "[选择:time-6h] 6小时"

2. 发送: "继续"

**观察点**:
- AI 是否说 "你没选" 或 "等你选择"？
- AI 是否继续推进流程？

**预期结果**: AI 正确识别已做出的选择，不重复提问。

---

### 2.5 测试场景 5: 阶段推进

**测试步骤**:

对每个阶段（可以在代码中设置或自然推进）:

1. 设置当前阶段（如 stage_04_track）
2. 发送: "请给我这个阶段的指导"

**观察点**:
- AI 回复是否包含阶段相关关键词？
  - stage_01_brainstorm: "脑爆"、"选题"
  - stage_04_track: "技术"、"轨道"
  - stage_07_execute: "代码"、"开发"
- 回复是否有意义？

---

### 2.6 测试场景 6: 教学模式

**测试步骤**:

1. 进入 stage_07_execute 阶段
2. 切换到不同教学模式:
   - `guided`: 引导式
   - `demo`: 演示式
   - `hands_on`: 动手式
   - `lecture`: 讲解式

3. 请求: "教我写一个简单的计算器"

**观察点**:
- **guided**: 是否提供代码框架和 TODO？
- **demo**: 是否提供完整可运行的代码？
- **hands_on**: 是否给出挑战任务？
- **lecture**: 是否解释原理？

## 5. 常见问题排查

### 5.1 后端连接失败

**现象**: 测试报错 `response.ok() is false`

**排查**:
```powershell
# 检查后端是否运行
curl http://localhost:3200/api/v1/health

# 检查端口占用
netstat -ano | findstr :3200
```

### 5.2 AI 回复为空或异常

**排查**:
```powershell
# 检查 ZeroClaw Gateway
curl http://127.0.0.1:42617/health

# 检查后端日志
tail -f apps/backend/logs/app.log
```

### 5.3 选项卡不显示

**排查**:
- 检查浏览器控制台是否有 JavaScript 错误
- 检查 WebSocket 连接是否正常
- 检查后端是否正确返回 ask_question 工具调用

### 5.4 选择不被识别

**排查**:
- 检查发送的消息格式是否为 `[选择:选项ID]`
- 检查后端 `_parse_selection_format` 函数
- 检查 `orchestrator.py` 中的跟进逻辑
