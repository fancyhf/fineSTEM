## 1. 测试范围与目标

### 1.1 测试目标
验证fineSTEM平台"创造"页面的AI对话流全流程功能，确保：
- Skill选择器正常工作
- AI生成代码自动流入编辑器
- 项目正确创建并持久化到后端数据库
- 代码预览功能正常工作
- 历史项目可查看和切换

### 1.2 测试范围
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **Skill选择器** | 默认Skill、切换Skill、UI展示 | P0 |
| **AI对话** | 发送消息、流式响应、代码提取 | P0 |
| **项目创建** | 点击新建项目、AI触发建项目、后端API调用 | P0 |
| **代码编辑器** | 自动打开、代码流入、语言切换 | P0 |
| **代码预览** | 运行按钮、HTML/JS/Python预览、srcDoc渲染 | P0 |
| **项目列表** | 左侧显示当前项目、历史项目加载 | P1 |

### TC-005: 项目创建 - 后端API调用验证

**优先级**: P0 | **前置条件**: 用户已登录 | **预计耗时**: 3min

#### 测试步骤
1. 登录系统获取token
2. 在Create页面输入："我想做一个天气查询应用"
3. 发送消息并等待AI响应
4. 打开浏览器DevTools → Network标签
5. 查找POST请求到 `/api/v1/projects`
6. 同时观察左侧"**项目**"区域

#### 预期结果（两种路径都应满足）

**路径A：AI调用project_creator工具**
✅ Network中存在 `POST /api/v1/projects` 请求
✅ 请求体包含：name, mode, author_id
✅ 响应状态码 201 或 200
✅ 响应体包含 project_id 和 name
✅ 左侧"**项目**"区域显示：📍 [项目名称]
✅ 显示模式（light/standard）和当前阶段

**路径B：前端Fallback机制（当AI未调用工具时）**
✅ 如果AI未调用工具，前端自动创建本地项目
✅ 左侧"**项目**"区域仍显示项目信息
✅ 控制台日志显示："Fallback: 创建本地项目"

#### 后端验证（附加）
```bash
# 通过API检查项目是否真的创建了
curl http://localhost:8001/api/v1/projects \
  -H "Authorization: Bearer {token}" | jq '.data[] | select(.name | contains("天气"))'
```
✅ API返回的项目列表包含刚创建的项目
