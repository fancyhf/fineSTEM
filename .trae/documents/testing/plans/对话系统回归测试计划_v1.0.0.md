**版本**: v1.0.0

> **本计划是对话系统质量的最终把关**。覆盖选项卡渲染、多选、多卡、重复问、总结误识别等全部历史问题。

## 0. 背景：为什么必须推进到 stage_04+

问题清单中的问题不是在单一阶段出现的——它们分布在不同阶段：

| 阶段 | 覆盖的问题 | 说明 |
|------|-----------|------|
| stage_00 | Q-001 丢卡、Q-002 工具调用、Q-010 [选择] 格式 | 3 轮连续提问是丢卡的高发期 |
| stage_01 | Q-005 重复问、Q-006 多选 | 脑爆阶段有多选场景（兴趣爱好） |
| stage_02 | Q-007 只总结无下一步、Q-003 总结误识别 | 开题阶段 AI 倾向总结 |
| stage_03~04 | Q-004 重复卡 | 跨阶段推进时可能重复入栈 |

**只测前 2 轮（stage_00）无法发现 stage_02 的总结误识别和 stage_01 的多选问题。**

## 1. 强制要求

### 1.1 必须有头测试

```bash
# Playwright 必须用 --headed（有头浏览器），不用 headless
cd apps/frontend
RUN_AI_E2E=1 npx playwright test --project=chromium --headed --video=retain-on-failure --screenshot=on
```

**理由**：
- headless 模式下无法截图取证，出问题时无法复现
- 有头模式可以观察渲染时序，发现间歇性丢卡
- 录屏保留失败现场的完整交互过程

### 1.2 必须推进到 stage_04

测试脚本必须自动完成 stage_00 → stage_01 → stage_02 → stage_03 → stage_04 的完整流程。
每个阶段都要点选项卡回答，验证卡片渲染和多选。

### 1.3 必须对照问题清单

测试报告必须逐项回答 Q-001~Q-010 的状态（✅通过 / ❌复现 / ⚠️未覆盖）。

## 3. 执行顺序

```
0. 前置准备
   - 重启 ZeroClaw daemon（加载最新 prompt）
   - 启动前端 dev server（加载最新前端代码）
   - curl 确认两个服务健康

1. 单元测试（TC-DLG-011~020）
   cd apps/frontend && npx vitest run
   cd apps/frontend && npx tsc --noEmit
   cd apps/frontend && npx vite build
   cd apps/backend && python -m pytest tests/test_mcp_server.py tests/test_stage_constants.py tests/test_tools_gates.py tests/test_check_gate_structural.py -v

2. WS 回归测试（TC-DLG-021~025）
   cd apps/backend
   set PYTHONIOENCODING=utf-8
   python scripts/ws_regression_test.py

3. ⚠️ Playwright 有头 E2E（TC-DLG-001~010，最核心）
   cd apps/frontend
   set RUN_AI_E2E=1
   npx playwright test --project=chromium --headed --video=retain-on-failure --screenshot=on

4. 问题清单对照
   → 逐项确认 Q-001~Q-010 状态
   → 测试报告中列出每项结果

5. 出测试报告
   → reports/对话系统回归测试报告_<date>.md
```

## 4. Playwright spec 要求

测试脚本（`apps/frontend/tests/specs/zeroclaw-integration.spec.ts`）必须：

1. **有头模式**：`--headed` + `--video=retain-on-failure` + `--screenshot=on`
2. **推进到 stage_04**：自动完成 stage_00 三轮 → stage_01 脑爆 → stage_02 开题 → stage_03 范围 → stage_04 轨道
3. **每轮验证**：检查 `question-card` 可见 + 点选项 + 等下一轮
4. **多选验证**：stage_01 的兴趣卡片选 2 个选项
5. **总结误识别验证**：stage_02 发"总结进度"后检查无 `question-card`
6. **截图取证**：每轮 before/after 截图

### 关键 testid

| testid | 元素 | 用途 |
|--------|------|------|
| `chat-input` | 聊天输入框 | 输入消息 |
| `send-button` | 发送按钮 | 点击发送 |
| `question-card` | 选项卡根元素 | 验证卡片出现 |
| `question-option` | 选项按钮 | 点击选项 |
| `message-assistant` | AI 消息 | 验证回复内容 |

## 6. 环境准备

```bash

powershell: Stop-Process -Name zeroclaw -Force

# 启动新 daemon（加载最新 config.toml + SKILL.md + SOUL.md）
```
