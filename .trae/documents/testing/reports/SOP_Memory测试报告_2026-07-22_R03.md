## 四、问题清单

### 问题 #001 — TC-SM-WS-003 SOP WebSocket 测试超时

- **用例**: TC-SM-WS-003 SOP 工具触发
- **严重度**: 严重
- **现象**: 发送"请执行 pbl-stage-flow SOP"消息后，AI 回复"好的，我来执行 PBL 流程！让我先读取 SOUL.md 和 USER.md，然后调用 SOP。"，随后 120s 超时，未收到任何 sop_execute tool_call 事件。
- **复现步骤**:
  1. 确保 ZeroClaw daemon 已重启并加载新配置
  2. `cd apps/backend && set PYTHONIOENCODING=utf-8 && python scripts/ws_sop_test.py`
  3. 观察输出：握手成功 → 发送消息 → 120s 超时 → FAIL
- **期望行为**: AI 应直接调用 sop_execute 工具启动 SOP 运行，tool_result 返回 run_id
- **日志**: `logs/2026-07-22/ws-sop-test-r03.log`
- **初步判断**: 
  - **非配置问题**：SOP 定义已正确加载（`sop validate` 通过，`sop list` 含 pbl-stage-flow），auto_approve 含 sop_execute
  - **AI 模型行为问题**：DeepSeek 模型在收到"执行 SOP"指令后，尝试先读取 SOUL.md 和 USER.md 文件（通过 shell/file_read 工具），这些文件操作可能耗时过长或卡住，导致 120s 内未完成 sop_execute 调用
  - 可能的根因：system_prompt 中未明确指示 AI 在收到 SOP 执行请求时应直接调用 sop_execute 工具，而非先读取文件
- **修改建议**:
  1. 在 system_prompt 中增加明确指令："当用户请求执行 SOP 时，直接调用 sop_execute 工具，无需先读取 SOUL.md/USER.md"
  2. 或在 ws_sop_test.py 中调整消息措辞，更明确地指示 AI 直接调用工具
  3. 或增加 RESPONSE_TIMEOUT 到 180s，给 AI 更多时间完成前置操作
  4. 考虑在 config.toml 的 [agents.assistant] 中检查是否有 SOUL.md/USER.md 引用导致 AI 分心
- **状态**: 待开发确认
