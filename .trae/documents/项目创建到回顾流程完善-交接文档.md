## 1. 项目概况

fineSTEM 是一个 STEM PBL（项目制学习）平台。核心流程：

```
Create（聊天+代码编辑器）→ Research（项目列表）→ ProjectDetail（9 阶段详情+导出）
                                                      ↓
                                              ProjectAchievement（成果档案卡）
```

**技术栈**：FastAPI 后端（`apps/backend`）+ React/Vite 前端（`apps/frontend`）。

## 3. 我新增的改动（仅 1 个文件）

### 3.1 `apps/frontend/src/pages/ProjectDetail.tsx` — `?file=` 参数消费

**上游触发**：Create 聊天中 AI 说 `报告保存在 docs/reports/final.md`，MarkdownText 把它渲染成
`<Link to="/research/projects/{projectId}?file=docs/reports/final.md">`

**我的改动**：
- 引入 `useSearchParams`，读取 `?file=`
- 调用 `projectsApi.getWorkspace(id)` 拿 `workspace.filename`
- 如果文件名匹配（近似匹配：完全相等 / 路径后缀匹配）→ 显示"进入代码编辑器"按钮
- 否则 → 显示"项目完整资料请导出 ZIP"
- 有关闭按钮（sessionStorage 记一次关闭）

**但这里有一个验证缺失**：getWorkspace 是异步的，文件名匹配逻辑只在 workspace 返回后才生效。没有 loading 态处理。

## 4. 已知断点 & 真实问题

### 4.1 Research 页面「已完成」tab 的链路未闭环

- **按钮代码已改**：两个 tab 都有"项目详情"按钮
- **但实际体验**：点击已完成项目的"查看/生成成果卡" → 跳转到 `/research/projects/:id/achievement`
- **如果 achievement 不存在**：ProjectAchievement 会显示"还没有成果档案卡"引导页 → 点"去项目详情生成"回到 ProjectDetail
- **但是 ProjectDetail 里的「生成成果档案卡」按钮**：设置 `showAchievementForm=true` 后渲染的是**内联的 `<ProjectAchievement>` 表单组件**（不是路由导航），用户在 ProjectDetail.tsx 内填写表单 → 创建 achievement → 卡片出现在右侧预览区
- **闭环缺失**：创建成功后没有自动跳转到 `/research/projects/:id/achievement` 来展示完整档案卡+分享功能。用户需要手动再点一次"查看成果档案卡"按钮。

### 4.2 ProjectDetail 的代码质量问题（预存）

`lint` 报 1 个 error + 5 个 warning：
- **Error**（line 40）：`loadProjectData` 在声明前被 useEffect 调用（`react-hooks/immutability`）。虽然是合法的 JavaScript hoisting，但 eslint 视为问题。**不建议大改（避免引入更多 bug）**。
- **Warning**（line 21）：`user` 变量已声明但未使用
- **Warning**：4 处 `any` 类型

### 4.3 StandardProjectSteps 和 EvidencePanel 有 lint error（预存）

这两个文件有 `react-hooks/set-state-in-effect` 的 lint 错误，是前任代码自带的，不影响编译。但如果跑 `lint --max-warnings 0` 会失败。

### 4.4 成就卡与项目详情的导航混乱

存在**两套 ProjectAchievement**：
1. **路由级**：`/research/projects/:id/achievement` → `src/pages/ProjectAchievement.tsx`（独立页面，含分享链接、灵感墙发布、下一步推荐）
2. **内联级**：ProjectDetail.tsx 内定义的 `<ProjectAchievement>` 组件（表单而已，仅创建卡片）

从 Research 页面的"查看/生成成果卡"按钮跳的是**路由级**。但创建成就卡是从 ProjectDetail 的**内联表单**创建的。这是两条不同路径，新用户会绕晕。

## 5. 深层问题（最该优先排查）

### 5.1 Create 聊天里 projectId 可能是 null

`Create.tsx` line 1614：
```tsx
projectId={projectContext.projectId?.startsWith('local-') ? null : projectContext.projectId}
```

分析：
- 刚进入 Create 页面、还没创建项目时 → `projectContext.projectId` = `null` → MarkdownText 收不到 projectId → AI 提到的文件路径**不会**变成可点击链接
- 即使`ensureProjectCreated`异步创建了项目 → 但**第一条 AI 回复流式输出时**，`projectContext.projectId` 可能还没更新 → 整条回复中的文件路径都不可点击
- 只有刷新或发第二条消息时，projectId 才对

**这大概率就是用户说的"点击起来缺失厉害"——第一轮对话里 AI 生成的文件路径压根不是链接。**

### 5.2 MarkdownText 文件路径匹配范围过窄

`src/components/MarkdownText.tsx` line 19-23：
```ts
function isProjectPath(text: string): boolean {
  return (
    /^(src|docs|reports|assets|code|data|tests|public|app|pages|components|server|backend|frontend)\//i.test(value) ||
    /^[\w./\-\u4e00-\u9fff]+?\.(html|css|js|jsx|ts|tsx|py|md|json|pdf|docx|pptx|zip)$/i.test(value)
  );
}
```

问题：
- 只匹配以特定目录开头或以文件扩展名结尾的路径
- AI 可能输出 `代码文件：index.html`、`请查看 ./output/main.py`、`文档位置：reports/final`（无扩展名）→ **都不匹配**
- 匹配规则没考虑中文标点（冒号后跟路径）

### 5.3 ProjectDetail 的 `?file=` 匹配逻辑不完善

我的改动：用 `workspace.filename` 和 `?file=` 做模糊匹配。但：
- workspace.filename 是用户最后一次"保存代码"时存的文件名（可能跟 AI 提到的路径不一致）
- AI 提到 `docs/reports/final.md` → 在 ZIP 里确实有这个文件，但 workspace.filename 是 `src/index.html` → 匹配失败 → 只能显示"请导出 ZIP"
- 没有做 evidence 列表的全文搜索匹配
