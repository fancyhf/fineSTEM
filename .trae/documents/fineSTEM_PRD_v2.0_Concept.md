## 1. 产品愿景与定位 (Vision & Scope)

### 1.1 产品愿景

从“编程工具”升级为\*\*“青少年 AI PBL (Project-Based Learning) 项目式编程伴学体系”\*\*。
通过 AI Agent 扮演“科学导师 + 企业技术专家”的双重角色，引导学生经历完整的软件工程与科研流程，实现从“写代码”到“解决问题、创造价值、产出成果”的跨越。

### 1.2 核心价值主张

1. **全流程 PBL**: 覆盖选题、调研、方案设计、开发实施、测试发布、结案答辩全链路。
2. **Agent 驱动**: AI 不仅生成代码，更负责流程引导、文档辅助、思路启发（苏格拉底式教学）。
3. **成果导向**: 强制产出高价值成果（PPT、技术文档、论文、综评素材），服务于升学与竞赛。
4. **分级成长**: 针对不同年龄段认知特点，提供阶梯式的工具与内容支持。

## 3. 核心业务流程：AI PBL 七步法 (The PBL Workflow)

本流程适用于中高年级，由 AI Agent 全程引导。

### Phase 1: 引导式选题 (Inception)

* **目标**: 确立具有时代视野、科技含量且符合学生背景的项目主题，**重点实现与学校课程（数学、科学、人文等）的深度融合**。

* **AI 行为**:

  * **对话引导**: “你最近关注什么新闻？”“学校里有什么让你觉得麻烦的事？”**“最近在学校里学了什么有趣的知识点（如物理的抛物线、地理的洋流）？”**

  * **选题过滤器**: 匹配 PISA/NGSS 标准**及校内课程大纲**，推荐方向：数据可视化、互动媒体、实用工具、科学模拟。**确保选题既能锻炼编程，又能巩固学科知识（如“用 Python 模拟孟德尔豌豆杂交实验”）。**

* **产出物**: 《项目立项单》（含项目名称、背景、解决的问题）。

### Phase 2: 需求与方案设计 (Analysis & Design)

* **目标**: 培养系统思维，将模糊想法转化为具体方案。

* **AI 行为**:

  * **苏格拉底提问**: “这个功能是为了谁设计的？”“如果数据量很大怎么办？”

  * **文档辅助**: 辅助生成思维导图、流程图描述。

* **产出物**:

  * **《项目方案 PPT》**: 用于演示构思。

  * **《需求规格说明书 (PRD 简化版)》**: 定义功能列表、界面草图。

### Phase 3: 知识准备与 CS 课程 (Learning)

* **目标**: “即时学习 (Just-in-Time Learning)”，缺什么补什么。

* **功能**: **半定制 DIY 课程系统**。

  * **基础**: 基于预设知识图谱（Python 基础、数据结构、算法）。

  * **定制**: 学生提出需求（“我要做人脸识别”），AI 实时生成针对性微课/Demo 代码。

### Phase 4: 开发与实现 (Implementation)

* **目标**: 编写代码，解决技术难题。

* **技术栈**: Python (核心), Web 前端, C/C++ (算法/底层)。

* **AI 行为**:

  * **代码生成**: 模块化生成代码。

  * **调试引导**: 解释报错原理，而非直接给出修复后的代码块。

* **产出物**:

  * **源代码**: 可运行的项目工程。

  * **《技术实现文档》/《小论文》**: 记录核心算法、遇到的困难及解决方案（技术复盘）。

### Phase 5: 测试与数据分析 (Testing & Data)

* **目标**: 验证成果，体现科学性。

* **Web 端特性**: 支持 CSV/JSON 上传，利用 Python (Pandas/Matplotlib) 进行数据处理与可视化。

* **产出物**: 测试报告、数据分析图表。

### Phase 6: 发布与展示 (Release)

* **目标**: 成果社会化。

* **AI 行为**: 辅助编写 Readme，生成演示视频脚本，指导部署到 Web/Gitee。

* **产出物**: 在线 Demo 链接、演示视频。

### Phase 7: 结案与评价 (Conclusion)

* **目标**: 沉淀成果，服务升学/综评。

* **产出物**:

  * **《结案报告》**: 完整的项目复盘。

  * **《研学论文》**: 符合学术规范的格式。

  * **综评素材包**: 导出为 PDF，直接用于综合素质评价系统填报。

## 4. 功能模块详述 (Functional Requirements)

### 4.1 通用能力：AI Spec Prompt 体系

* **定义**: 一套跨平台的 Prompt 规则库与 Agent 编排逻辑。

* **适用**: 高年级 (IDE) 直接调用，中年级 (Web) 封装在 UI 后。

* **核心组件**:

  * `Role_Prompts`: 定义导师角色（严谨、启发式）。

  * `Phase_Prompts`: 对应 PBL 七步法的每个阶段。

  * `Doc_Gen_Prompts`: 专门用于生成 PPT 大纲、论文摘要、技术文档。

### 4.2 Web Agent 平台 (中年级核心)

* **架构**:

  * 前端: React/Vue 交互界面。

  * 后端: 支持轻量级数据存储 (SQLite/CSV)，提供 Python 运行时沙箱。

* **关键功能**:

  * **项目向导**: 步骤条形式展示 PBL 进度。

  * **数据工坊**: 内置简易 Excel 视图，支持上传 CSV，一键生成分析图表。

  * **知识卡片**: 侧边栏根据当前代码上下文推荐 CS 知识点（半定制课程入口）。

### 4.3 IDE Agent 插件 (高年级核心)

* **集成**: 适配 Trae/VSCode。

* **特性**:

  * **工程感知**: 自动读取整个项目文件结构。

  * **规范约束**: 强制要求代码符合 PEP8 等规范。

  * **算法衔接**: 针对 C/C++ 算法题提供专门的调试视图与复杂度分析。

### 4.4 半定制课程系统

* **逻辑**: "预设图谱 + AI 动态生成"。

* **场景**:

  1. **基础路径**: 系统预置的 Python 语法、数据结构课件。
  2. **按需生成**: 学生询问 "如何实现文字转语音？"，系统基于图谱检索相关库，并由 AI 实时生成一段带注释的 Demo 代码和原理解析。

## 5. 数据与技术要求 (Technical Constraints)

* **数据存储**:

  * Web 端必须支持用户上传轻量级数据集 (CSV, JSON, Txt)。

  * 支持基本的 SQL 查询或 Pandas DataFrame 操作。

* **文档生成**:

  * 需集成 Markdown 转 PDF/PPT 的能力（或通过 Pandoc 服务）。

* **兼容性**:

  * Web 端需适配主流平板与 PC 浏览器。

  * IDE 插件需跟随主流编辑器版本更新。

## 6. 附录：文档产出标准示例

* **项目方案 PPT**: 封面、背景(Why)、目标(What)、计划(How)、预期成果。

* **技术论文**: 摘要、关键词、正文(引言/方法/实验/结果)、参考文献。

* **结案报告**: 项目截图、功能清单、代码统计、个人感悟。
