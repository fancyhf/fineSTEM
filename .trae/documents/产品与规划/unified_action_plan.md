## 1. 战略顶层设计：重新定义“编程学习”

我们不是在做一个“编程工具”，也不是在做一个“在线课程”，而是在做一个\*\*“基于AI的数字化PBL（项目式学习）生态”\*\*。

* **核心理念**: **AI编程伴学 (AI-Assisted Learning Companion)**

* **价值主张**: **“把黑盒变成透明盒子”** —— AI不再只是代写代码的工具，而是通过解释代码逻辑，让学生看到“魔法”背后的原理。解决家长“孩子是否在偷懒”的顾虑。

* **教学法**: **Just-in-Time Learning (即时学习)** —— 不在项目开始前灌输语法，而在项目进行中，当学生遇到问题时，AI适时切入讲解语法、算法或学科知识。

***

## 2. 统一产品矩阵 (Unified Product Matrix)

为了同时满足“短视频传播”和“深度学习”的需求，我们并行推进两种形态，并覆盖**五大成长赛道**：

### 2.0 核心能力模型：五大成长赛道 (The 5 Tracks)

借鉴自《AI+创意学习体系》，我们为不同兴趣的学生设计了五条进阶路径：

* **Track A (Algorithms)**: 算法与逻辑（适合数学好的学生，对接 OI/竞赛）。

* **Track B (Product/Hackathon)**: 产品与工程（适合喜欢做应用的学生，对接黑客松）。

* **Track C (Social Good)**: 公益与同理心（适合关注社会的学生，做老人助手、环保App）。

* **Track D (Research)**: 科学探究（适合喜欢写论文/做实验的学生）。

* **Track E (Data Analysis)**: 数据分析与洞察（适合喜欢图表、统计的学生）。

### 形态 A：Web 自适应交互终端 (The "Hook")

* **定位**: **极简体验、流量承接、短视频转化**

*   **技术形态**: Web App (React + AI SDK)，自适应移动端/PC端。
*   **交互模式**: **响应式布局**。
    *   **PC端**: 经典“左侧预览 + 右侧对话”分栏。
    *   **移动端**: 采用“上下分布”或“悬浮抽屉/气泡”，优先保证主功能区（预览/图表）的可视面积。
*   **使用场景**:
    *   **物理/数学**: 修改参数改变物理世界（Track D）。

  * **数据分析**: 上传 Excel/CSV -> AI 清洗数据 -> 自动生成图表 -> 讲述数据背后的故事（Track E）。

* **用户路径**: 用户在抖音/B站刷到视频 -> 点击评论区链接 -> 30秒内体验“修改参数改变物理世界”或“一键分析成绩单”的快感。

### 形态 B：IDE Agent 伴学系统 (The "Core")

* **定位**: **沉浸式PBL平台、半成品项目工厂**

* **技术形态**: 基于 VS Code / Trae 的插件或配置包 (`.cursorrules` / `.trae/rules`)。

* **核心机制**:

  * **Idea Generator**: AI 引导学生头脑风暴，生成个性化项目 Idea。

  * **Starter Kits (半成品包)**: 提供 60% 完成度的代码，学生负责关键逻辑修改和创意发挥。

  * **Interactive Comments (埋点式注释)**: 在代码关键处预埋引导性注释（如 `// TODO: 试着把 g=9.8 改成 -9.8，问问 AI 会发生什么？`），触发主动探索。

  * **Contextual Tutor (上下文导师)**: AI 识别当前代码上下文，讲解背后的算法（如排序、递归）或学科知识（如牛顿定律）。

  * **AI Programming Companion Agents (AI编程伴学Agent组)**:

    * **苏格拉底导师 (The Socratic Tutor)**: 绝不直接给出代码答案，而是通过提问引导学生自己思考（如“你觉得这里为什么报错？”）。

    * **代码审计员 (The Code Reviewer)**: 模拟严格的 Tech Lead，检查代码规范和潜在 Bug，解释“为什么要这样写”。

    * **创意缪斯 (The Creative Muse)**: 当学生卡壳时，提供发散性的灵感（如“如果给小球加个拖尾效果会怎样？”）。

    * **学科专家 (The Domain Expert)**: 专门负责将代码逻辑关联到课本知识点（如“这段循环其实就是数学里的求和符号 Σ”）。

    * **数据分析导师 (The Data Analyst)**: \[Track E专属] 专注于教学生“清洗数据”、“选择图表”和“解读趋势”，而非仅仅生成代码。

## 3. 项目与场景库规划 (PBL Scenario Library)

我们的核心资产是\*\*“PBL项目库”\*\*。每个项目都是一个独立的交付物。
**选题方法论**: 拿着初高中课本找痛点（如物理实验数据处理、化学方程式配平、历史年代记忆）。

### 3.1 学科融合类 (Subject Integration) - Tracks A/D/E

* **物理模拟器 (MVP首选)**:

  * *项目*: “反重力实验室”（基于抛物线代码）。

  * *交互脚本*: 学生将重力 `g=9.8` 改为 `-9.8` -> 球向上飞 -> AI 解释：“你创造了反重力场！在代码宇宙中，负号代表方向相反。”

  * *知识点*: 变量赋值、坐标系（编程）；重力加速度、向量方向（物理）。

* **历史侦探 (跨学科经典)**:

  * *项目*: **“丝绸之路数据复原计划”**（参考自《历史跨学科教案》）。

  * *场景*: 提供一份模糊的古籍文本（《马可·波罗游记》片段） -> 学生使用 AI 提取地名和特产 -> AI 辅助绘制古代贸易地图。

  * *知识点*: 文本清洗、词频统计、地图可视化（Track E）；历史地理、文献考据（历史）。

* **数学可视化**:

  * *项目*: 蒙特卡洛方法算圆周率。

  * *知识点*: 随机数生成、循环结构（编程）；概率统计（数学）。

### 3.2 实用工具类 (Productivity Tools) - Track B

* **学业辅助 - 成绩分析师 (强痛点)**:

  * *项目*: “我的考试雷达图”。

  * *场景*: 输入几次考试成绩 -> AI 生成趋势图和雷达图 -> 预测下次及格率。

  * *知识点*: 数据结构、图表库（编程）；统计分析（数学）。

* **学业辅助 - 词汇小帮手**:

  * *项目*: **“自动单词卡生成器”**。

  * *场景*: 输入一篇英语课文 -> AI 提取生词、音标和例句 -> 自动生成可打印的单词卡 PDF。

  * *知识点*: 字符串处理、API调用（编程）；英语词汇（英语）。

### 3.3 创意娱乐类 (Creative Fun) - Track C/B

* **游戏开发**:

  * *项目*: 复刻 Flappy Bird（换皮版，把鸟换成班主任头像）。

  * *知识点*: 游戏循环、事件监听（编程）。

* **极客整活**:

  * *项目*: 表情包生成器（给图片加字）。

  * *知识点*: 图像处理库（编程）。

### 3.4 智能硬件类 (AIoT)

* **项目**: 树莓派/ESP32 智能温湿度计。

* **知识点**: 串口通信、传感器读取（编程）；电路基础（物理）。

## 5. 全程实施路线图 (Execution Roadmap)

### 第一阶段：MVP 爆破期 (Month 1) - 聚焦 Track A & E

* **目标**: 跑通 **“一个视频 + 一个Demo”** 的最小闭环，验证流量。

* **核心策略**: 利用“物理反直觉”（Track A）和“数据可视化”（Track E）作为两大流量抓手。

* **任务**:

  * \[产品] 开发 **“反重力实验室”** (Web版 MVP)，包含预设的 AI 引导语。

  * \[产品] 开发 **“成绩单雷达图”** (Web版 MVP)，支持 Excel 上传与自动分析。

  * \[媒体] 制作 **“别死记公式了，看我把它画出来”** 短视频。

  * \[基建] 搭建基础的 Web 托管环境和 AI 对话接口。

### 第二阶段：IDE 体系建设期 (Month 2-3) - 拓展 Track B & C

* **目标**: 沉淀深度用户，建立伴学模式。

* **任务**:

  * \[产品] 定义 IDE Agent 的人设 Prompt (`.trae/rules`)，开发包含“埋点注释”的半成品项目包。

  * \[内容] 推出 **“AI 编程第一课”** 系列视频教程，覆盖“单词卡生成器”（Track B）。

  * \[运营] 建立首个“AI 创造营”社群，招募种子用户。

### 第三阶段：平台化与生态期 (Month 4+) - 全赛道开放 (Tracks A-E)

* **目标**: UGC 内容涌现，形成壁垒。

* **任务**:

  * \[平台] 上线 Web 端“我的项目库”，支持学生分享作品。

  * \[活动] 举办 **“AI 黑客松 (Hackathon)”**：主题“为你的学习生活减负”，鼓励用 AI 解决实际问题。

  * \[赛道] 正式推出 **“历史侦探”** 等跨学科深度项目（Track D）。
