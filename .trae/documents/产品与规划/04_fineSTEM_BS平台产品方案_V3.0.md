## 1. 产品定位

### 1.1 一句话定义

**面向青少年的 STEM 研学助手平台——在线体验、AI 引导、过程驱动，从兴趣到论文一站式。**

### 1.2 与历史定位的演进

| 版本 | 定位 | 核心形态 | AI 角色 |
|------|------|---------|---------|
| v1.0 (白皮书) | "不用背语法，也能让计算机听你的话" | 媒体驱动 + Web终端 + IDE插件 | 苏格拉底导师 |
| v2.0 (PBL体系) | "AI PBL 项目式编程伴学体系" | Web Agent + IDE Agent | 流程引导 + 多角色Agent |
| **v3.0 (本方案)** | **"青少年 STEM 研学助手平台"** | **B/S 平台 + ZeroClaw AI 运行时 + AI IDE 互联** | **ZeroClaw Skill 驱动，过程引导，研学闭环** |

### 1.3 核心差异：不是编程平台

```
传统编程平台（豆包编程/Scratch/Code.org）：
  学生 → 学编程 → 做作品 → 展示

fineSTEM v3.0：
  学生 → 发现兴趣 → 立项（工单）→ 研究过程（AI引导）→ 产出（论文/报告/作品）
         ↑                                                    │
         └──────────── AI 助手全程陪伴引导 ←────────────────────┘
```

传统平台教"怎么写代码"，fineSTEM 教"怎么做一个研究项目"——代码只是工具之一。

### 1.4 核心价值主张

1. **过程驱动**：不是只看结果，而是引导完成"选题→开题→开发→结题→论文"全流程
2. **AI 陪伴**：ZeroClaw 驱动的 AI 助手，不是冷冰冰的问答，是懂你的研学导师
3. **成果导向**：最终产出研学报告、小论文、项目作品——可用于综评、升学、竞赛
4. **开放互联**：与 Trae/Cursor/VSCode 等 AI IDE 互通，不是封闭平台

## 2. 用户模型

### 2.1 核心用户：学生（唯一主要操作者）

学生按参与深度自然分为三级，**不是硬性分组，而是同一平台的不同使用方式**：

```
Level 1 体验者 —— "我来看看"
├── 浏览 Demo 墙，试运行现有项目
├── 融合式学习 Demo（学语文的古诗词卡、学数学的数据可视化）
├── 看代码 + 问 AI 助手"这段代码什么意思"
├── 零门槛，注册即用
└── 典型场景：课间10分钟体验一个物理模拟

Level 2 创作者 —— "我想做一个"
├── 在线选题开题（AI 引导脑爆→开题流程）
├── 简单代码在线编辑运行（沙箱环境）
├── 复杂项目：平台生成项目包，下载到本地用 AI IDE 开发
├── 研学流程全程在线追踪（工单、阶段检查点）
└── 典型场景：花6小时做一个待办清单，产出开题报告+代码+结题报告

Level 3 开发者 —— "我要深度开发"
├── 平台作为索引和参考站（找案例、找资料、找灵感）
├── 下载 fineSTEM Skill 安装到 AI IDE（不限于 Trae）
├── 项目成果可上传回平台展示
├── AI 助手在线咨询（不限于当前项目上下文）
└── 典型场景：用 Trae 做一个 Kaggle 数据分析项目，用到 stem-pbl-guide Skill
```

### 2.2 次要用户

| 角色 | 使用方式 | 权限 |
|------|---------|------|
| 老师 | 查看学生进度、浏览案例库 | 只读 |
| 家长 | 查看学习报告 | 只读 |
| 管理员 | 管理内容、Demo、课程 | 全部 |

**不需要老师介入教学流程**——AI 助手就是导师。

## 3. 功能模块

### 3.1 模块全景

```
┌──────────────────────────────────────────────────────────────┐
│                     fineSTEM 平台                             │
├────────────┬─────────────────┬───────────────┬───────────────┤
│  探索中心   │   AI 工作台      │  研学流程      │  AI IDE 互联  │
│  (Explore) │   (Create)      │  (Research)   │  (Connect)    │
├────────────┼─────────────────┼───────────────┼───────────────┤
│ Demo 墙    │ AI 对话面板      │ 我的项目       │ Skill 市场    │
│ 课程库     │ 在线代码编辑器   │ 研学工单       │ 项目导入导出  │
│ 灵感墙     │ 项目下载中心     │ 研学文档       │ 远程 AI 咨询  │
└────────────┴─────────────────┴───────────────┴───────────────┘
```

### 3.2 探索中心 (Explore)

**定位**：Level 1 体验者的主入口，所有学生的发现层。

#### 3.2.1 Demo 墙

- **项目卡片网格**：封面图 / 名称 / 难度标签 / 学科标签 / 技术栈标签
- **一键体验**：
  - 纯 HTML/JS 项目：iframe 交互嵌入，可实时操作
  - Flask/Streamlit 等动态项目：静态化展示（截图 + 代码浏览），不能交互
- **代码查看**：只读模式，语法高亮，支持 AI 问答"这段代码什么意思"
- **下载项目包**：含 `start.bat` / `start.sh` + `README.md` + 源代码
- **初始项目**（已有）：
  - 文学知识卡（Flask，端口 4001）→ 静态展示
  - UP主视频分析器（Streamlit，端口 4002）→ 静态展示
  - 智能待办清单（纯 HTML/JS，端口 4003）→ iframe 交互嵌入
- **未来扩展**：项目会持续增加，Demo 墙支持动态添加

#### 3.2.2 课程库

- **融合式学习 Demo**：跨学科案例（语文+编程的古诗词卡、数学+编程的数据可视化）
- **按学科/难度/年级筛选**
- **每个课程关联 Demo + AI 问答入口**
- MVP 阶段：框架搭建 + 初始课程元数据

#### 3.2.3 灵感墙

- 展示其他学生完成的项目
- "我也能做"按钮 → 一键启动类似项目
- MVP 阶段：框架搭建，初始展示已有3个项目

### 3.3 AI 工作台 (Create)

**定位**：Level 2 创作者的核心工作区，Level 1/3 的辅助工具。

#### 3.3.1 AI 对话面板

- **自由对话**：问任何 STEM 问题
- **代码问答**：粘贴代码问"这段什么意思"、"为什么报错"
- **项目引导**：说"我想做一个项目" → 启动 StemPblGuide Skill
- **研学咨询**：问"怎么写开题报告"、"研究方法有哪些"
- **上下文感知**：AI 知道当前用户在做什么项目、处于哪个阶段
- **技术实现**：ZeroClaw Runtime 驱动，WebSocket 实时通信

#### 3.3.2 在线代码编辑器

- **编辑器**：Monaco Editor（VS Code 同款内核）
- **沙箱运行**：
  - JS：WebContainer（浏览器内运行，零服务器成本）— 优先实现
  - Python：Pyodide（浏览器内运行）— 后续实现
- **AI 辅助**：代码补全 / 代码解释 / 错误调试
- **代码分享**：生成分享链接 / 下载文件
- **限制**：仅支持简单代码运行，复杂项目引导到 AI IDE

#### 3.3.3 项目下载中心

- **生成项目包**：代码框架 + `docs/` + Skill 配置 + `README.md`
- **AI IDE 安装指引**：不限于 Trae，包括 Cursor / VS Code / CodeBuddy 等
- **Skill 安装教程**：如何将 fineSTEM Skill 安装到各种 AI IDE
- **一键下载 .zip**：包含所有启动所需文件

### 3.4 研学流程 (Research)

**定位**：fineSTEM 的核心差异化模块，过程驱动的研学引擎。

#### 3.4.1 我的项目

- **项目列表**：进行中 / 已完成
- **项目详情**：阶段进度条 + 工件状态 + 时间线可视化
- **项目操作**：新建 / 继续 / 导出 / 删除

#### 3.4.2 研学工单（核心差异化）

**设计原则**：引导但不强制。有过程性，但不是必须完成每一步。

- **阶段检查点**：非强制，但会提醒"建议完成"
- **工单模板**：复用 stem-pbl-guide 的 9 阶段状态机
- **AI 自动填充建议**：根据上下文推荐内容
- **证据上传**：截图 / 日志 / 数据文件

**9 阶段流程（移植自 stem-pbl-guide Skill）**：

| 阶段 | 名称 | 主要产出 | 门禁条件 |
|------|------|---------|---------|
| stage_00 | 初始化 | 项目目录 + SKILL_STATE.json | 文件存在 |
| stage_01 | 脑爆选题 | 脑爆记录 + 选题方向 | 至少1轮；候选≥6；有Top3 |
| stage_02 | 开题立项 | 项目立项书 | success_criteria≥2；risks≥2 |
| stage_03 | 范围裁剪 | 范围约束文档 | must_have≤3；wont_do≥2 |
| stage_04 | 轨道选择 | 技术轨道规划 | template_id 合法；资源不冲突 |
| stage_05 | 设计蓝图 | 设计方案 + 代码框架 | 验收用例≥3 |
| stage_06 | 分步计划 | 分步执行计划 | 每步有 run/check/rollback |
| stage_07 | 执行开发 | 源代码 + 开发日志 | 至少完成1个milestone |
| stage_08 | 验收展示 | 验收评估 | 验收项≥2；反思≥2 |

**灵活入口设计**：
- 学生可以说"我已经有想法了" → AI 帮他快速填完开题卡
- 学生可以说"我不想写论文" → paper_mode=off
- 学生可以说"我只想做个简单 Demo" → 跳过部分阶段
- 门禁保证最低质量底线，但路径和深度由学生决定

#### 3.4.3 研学文档

- **文档类型**：开题报告 / 技术报告 / 结题报告 / 论文（可选）
- **AI 辅助生成**：生成框架和引导，不是代写
- **教学模式**：4 种可选（引导式/演示式/动手式/讲解式）
- **导出格式**：PDF / DOCX / PPTX
- **证据素材**：截图 / 图表 / 日志 / 数据

#### 3.4.4 选题推荐

- **三轨道题库**（已有）：
  - Web 项目题库（20个，beginner~intermediate）
  - Kaggle/建模题库（10个，beginner~advanced）
  - 硬件创客题库（10个，beginner~advanced）
- **基于兴趣的 AI 推荐**
- **"我也能做"一键启动**

### 3.5 AI IDE 互联 (Connect)

**定位**：平台与本地 AI IDE 的桥梁，不是竞争关系，是接力关系。

#### 3.5.1 Skill 市场

- **fineSTEM Skill 列表**：stem-pbl-guide 等
- **安装指引**：不限于 Trae，支持 Cursor / VS Code / CodeBuddy 等
- **一键复制安装命令**
- **Skill 使用教程**

#### 3.5.2 项目导入/导出

- **平台 → 本地**：下载项目包（代码框架 + docs + Skill 配置）
- **本地 → 平台**：上传项目成果到灵感墙展示
- **状态同步**：SKILL_STATE.json 可在平台和 AI IDE 之间同步

#### 3.5.3 远程 AI 咨询

- 不在 AI IDE 里也能问 AI
- 项目上下文关联（AI 知道你在做什么项目）
- 代码片段粘贴咨询

### 4.2 技术选型原则

1. **适配 ZeroClaw**：后端技术栈选择与 ZeroClaw 的 API/绑定最自然的方案
2. **不考虑 SEO**：纯 SPA，无需 SSR/SSG
3. **在线体验优先**：代码查看、代码编辑、Demo 体验要流畅
4. **已有资产复用**：现有3个 Demo 项目、stem-pbl-guide Skill、MVP1 组件

### 4.3 ZeroClaw 集成策略

**分阶段集成**：先搭平台框架，再集成 ZeroClaw。

```
Phase 1: 平台框架
├── 前端 SPA 搭建
├── API Server 基础功能
├── Demo 墙（静态数据）
└── 用户系统

Phase 2: ZeroClaw 集成
├── 部署 ZeroClaw Runtime
├── 实现 WebSocket Tunnel
├── 移植 StemPblGuide Skill
└── AI 对话面板上线

Phase 3: 完整功能
├── 在线代码编辑器 + 沙箱
├── 研学流程 UI
├── 项目下载中心
└── AI IDE 互联
```

### 4.4 现有 Demo 项目集成方案

| 项目 | 技术栈 | 展示方式 | 代码查看 | 下载 |
|------|--------|---------|---------|------|

**未来新项目**：鼓励制作纯 HTML/JS 版本以支持交互嵌入；其他技术栈项目提供静态展示。

## 5. ZeroClaw Skills 规划

### 5.1 核心 Skills

#### StemPblGuide（研学流程引导）

- **来源**：移植自 `.trae/skills/stem-pbl-guide/`
- **功能**：9 阶段状态机 + 研学文档轨 + 4 种教学模式
- **适配**：从 Trae Skill 格式转换为 ZeroClaw Tool Trait
- **关键数据结构**：SKILL_STATE.json 保持兼容，平台和 AI IDE 可互操作

#### CodeRunner（在线代码执行）

- **功能**：安全执行学生代码，即时反馈
- **实现**：JS 通过 WebContainer，Python 通过 Pyodide
- **安全**：资源限制 + 超时控制 + 输出过滤

#### KnowledgeBase（知识检索）

- **功能**：fineSTEM 自有内容检索
- **数据源**：课程库 + 案例库 + 题库（topic_library_*.md / track_library.md）
- **检索方式**：向量嵌入 + 关键词搜索

#### DemoExplorer（Demo 体验引导）

- **功能**：解释 Demo 代码 + 引导"我也能做"
- **场景**：学生看 Demo 时问"这段代码什么意思"、"我想做一个类似的"

#### IDEConnector（AI IDE 互联）

- **功能**：生成项目下载包 + Skill 安装指引 + 项目状态同步
- **支持**：Trae / Cursor / VS Code / CodeBuddy 等
- **输出**：.zip 项目包（含代码框架 + docs + Skill 配置 + README）

### 5.2 Skill 开发原则

1. **自主开发**：fineSTEM 的 Skill 是核心竞争力，不依赖第三方 Skill 市场
2. **精简聚焦**：只做教育研学领域需要的 Skill，不做大而全
3. **平台/AI IDE 双跑**：同一 Skill 逻辑在 ZeroClaw（平台）和 Trae（本地）都可运行
4. **数据兼容**：SKILL_STATE.json 格式在平台和 AI IDE 之间完全兼容

## 8. 关键设计决策

### 8.1 引导强度：有过程性但不强制

- 门禁条件保证最低质量底线（如开题卡必须有 success_criteria）
- 学生可以跳过非关键阶段（如"我已经有想法了"可快速填完脑爆）
- AI 会建议但不强制（"建议你再想想其他方向？"而非"你必须完成脑爆"）
- 不需要老师介入，AI 助手就是导师

### 8.2 平台与 AI IDE 的关系：接力而非竞争

```
平台（起跑区）              AI IDE（赛道）
选题 → 开题 → 简单验证  →  深度开发 → 复杂代码
                          ↑
                    项目包下载 + Skill 安装
```

- 平台做轻量体验和流程引导
- AI IDE 做深度开发
- 两者通过项目包和 Skill 无缝衔接
- 不限于 Trae，支持任何 AI IDE

### 8.3 LLM 策略：纯云端

- 不做本地模型部署
- 通过 ZeroClaw Provider Trait 对接云端 LLM
- 支持 Claude / OpenAI / 国内模型等
- 通过 ZeroClaw 统一接口，切换模型无需改业务代码

### 8.4 Demo 展示策略：分层处理

- 纯 HTML/JS 项目：iframe 交互嵌入（最佳体验）
- 动态项目（Flask/Streamlit）：静态化展示（截图+代码浏览）
- 所有项目：支持代码查看 + 项目包下载
- 未来鼓励制作纯 HTML/JS 版本以支持交互嵌入

### 8.5 代码沙箱策略：渐进实现

- 第一优先：JS 通过 WebContainer（浏览器内运行）
- 第二优先：Python 通过 Pyodide（浏览器内运行）
- 不做服务器端 Docker 沙箱（成本和安全问题）
- 复杂代码引导到 AI IDE 本地运行

## 10. 风险与待决事项

### 10.1 待决事项

| # | 事项 | 影响 | 建议 |
|---|------|------|------|
| 1 | API Server 技术选型 | 开发效率 | 选择与 ZeroClaw 集成最自然的方案 |
| 2 | ZeroClaw Rust SDK 成熟度 | 集成难度 | 先用 JS/Python 绑定验证，Rust SDK 后续跟进 |
| 3 | WebContainer 兼容性 | 代码沙箱 | 验证主流浏览器支持情况 |
| 4 | Pyodide 包支持范围 | Python 沙箱 | 先支持纯 Python 标准库，第三方包逐步添加 |
| 5 | stem-pbl-guide 移植工作量 | 核心功能 | Skill 逻辑复用，交互层需重写为 Web UI |

### 10.2 风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| ZeroClaw 尚在早期，API 可能变化 | 中 | 高 | 抽象适配层，降低耦合 |
| 云端 LLM 成本 | 高 | 中 | 设置用量限制 + 缓存常见问答 |
| 代码沙箱安全 | 低 | 高 | WebContainer 天然隔离；Pyodide 限制文件系统访问 |
| 学生流失（Level 1 不转化） | 中 | 中 | 优化 Demo 体验 + "我也能做"引导 |
