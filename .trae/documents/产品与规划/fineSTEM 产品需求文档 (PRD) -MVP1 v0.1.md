## 1. 产品愿景 (Vision)

**“不用背语法，也能让计算机听你的话。”**

针对 7-10 年级中学生，打造一套**“AI编程伴学体系”**。利用 AI 大模型（LLM）作为核心引擎，将编程从“学习语法”转变为“学习逻辑与解决问题”。通过**短视频媒体传播**驱动用户认知，通过**嵌入式AI工具**提供沉浸式学习体验，最终实现学科融合与素养提升。

---

## 2. 核心概念：AI编程伴学 (AI-Assisted Coding Companion)

传统的编程教育是“老师教 -> 学生写 -> 报错 -> 老师改”。
我们的模式是**“提出目标 -> AI实现 -> 观察结果 -> 追问AI -> 修改迭代”**。

### 2.1 核心价值
1.  **学科融合 (Cross-disciplinary)**: 编程不是独立学科，而是解决物理、数学、英语问题的“外挂”。
2.  **即时反馈 (Instant Gratification)**: 消除语法报错的挫败感，通过AI解释和自动修复，保持学习心流。
3.  **苏格拉底式教学 (Socratic Method)**: AI 不直接给答案，而是通过预设的人设引导学生思考（需通过 Prompt Engineering 实现）。

## 3. 产品形态规划 (Product Form)

我们需要同时规划“工具”与“内容”，两者相辅相成。

### 3.1 形态 A：IDE Agent 伴学插件 (IDE Plugin/Config)
*   **定位**: 沉浸式、专业感强，适合有一定探索欲的学生。
*   **载体**: 基于 VS Code / Trae / Cursor 的插件或特定配置文件（`.cursorrules` 等）。
*   **核心功能**:
    *   **角色扮演**: 强制 AI 扮演“物理老师”或“游戏策划”，而非冷冰冰的代码助手。
    *   **上下文感知**: 自动读取当前打开的代码文件，回答时结合代码逻辑。
    *   **探索任务单**: 在 IDE 侧边栏提供“任务清单”，引导学生向 AI 提问（例如：“问问 AI 为什么这里的重力是 9.8？”）。

### 3.2 形态 B：Web 交互式自适应终端 (Web Adaptive Terminal)
*   **定位**: 零门槛、强交互、适合短视频引流和初学者体验。
*   **载体**: 网页端 (Web App)。
*   **核心功能**:
    *   **左侧**: 可视化运行窗口 (Canvas/HTML Preview)。
    *   **右侧**: 限制性 AI 对话框 (Guided Chat)。
    *   **特性**: 预置特定场景（如“抛物线模拟器”），学生只需通过对话修改参数，立刻在左侧看到变化。

## 4. 媒体与内容传播策略 (Media Strategy)

**原则**: 内容即产品，传播即教学。

### 4.1 短视频策划方向
*   **痛点唤醒类**: “作业太多写不完？看我用 Python 自动生成英语错题本。”
*   **降维打击类**: “别死记物理公式了，我让 AI 给我画出来了，动图一看就懂。”
*   **极客创造类**: “全班只有我的计算器能发表情包。”

### 4.2 叙事模型 (Storytelling)
*   **把代码包装成“魔法”**: 不讲 `import`，讲“召唤数学库”。
*   **把 Bug 包装成“挑战”**: 不讲 `SyntaxError`，讲“AI 没听懂你的咒语，换个说法试试”。

## 5. 实施路线图 (Roadmap)

### 第一阶段：概念验证与媒体启动 (MVP & Media Launch)
*   **目标**: 验证用户对“AI解决学科问题”的兴趣度。
*   **行动**:
    1.  **PRD 细化**: 确定首个演示案例（如：物理-抛物线 或 数学-统计图）。
    2.  **Demo 制作**: 手搓一个基于 IDE 的半成品项目，包含预设 Prompt。
    3.  **视频发布**: 录制“如何用 AI 搞定物理作业”的短视频，测试流量。

### 第二阶段：体系化课件与工具封装 (Curriculum & Tooling)
*   **目标**: 形成可复用的产品包。
*   **行动**:
    1.  **开发 IDE 插件/配置包**: 让学生下载即用。
    2.  **编写伴学文档**: 配套的 PDF/Notion 页面，指导学生如何向 AI 提问。
    3.  **建立社群**: 收集反馈，迭代 Prompt。

### 第三阶段：平台化 (Platform)
*   **目标**: 降低依赖，建立壁垒。
*   **行动**:
    1.  **开发 Web 端伴学工具**: 摆脱本地环境配置的门槛。
    2.  **UGC 社区**: 让学生分享自己用 AI 创造的工具。
