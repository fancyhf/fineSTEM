## 1. 附件目的

本附件用于回答 4 个执行层关键问题：

1. MVP 怎样才算验证成功
2. 学生是否可以匿名试玩，何时必须注册
3. 分享链接与灵感墙展示边界如何定义
4. 轻项目升级为标准研学时，数据如何映射

---

## 2. MVP 成功判据

### 2.1 核心判断

MVP 核心不是“功能做得多不多”，而是：

**学生是否愿意从 Demo 出发，在 AI 引导下完成一个轻项目，并生成可分享成果。**

### 2.2 核心指标

| 指标 | 定义 | 目标方向 |
|------|------|---------|
| Demo -> fork 点击率 | 试玩 Demo 后点击“我也做一个”的比例 | 越高越好 |
| fork -> 轻项目完成率 | 创建项目后完成轻项目并生成成果档案卡的比例 | 越高越好 |
| 成果档案卡生成率 | 创建项目后最终生成成果档案卡的比例 | 越高越好 |
| 成果分享率 | 生成成果档案卡后复制分享链接的比例 | 越高越好 |

### 2.3 最小成功条件

若以下三项同时成立，则可判定 MVP 主闭环成立：

1. 学生能从 Demo 进入自己的项目
2. 学生能完成至少一个轻项目
3. 学生愿意生成并分享成果档案卡

---

## 3. 匿名试玩与注册时机

### 3.1 原则

**试玩匿名化，保存注册化。**

### 3.2 规则

| 场景 | 是否要求注册 | 说明 |
|------|-------------|------|
| 浏览 Demo 墙 | 否 | 允许匿名浏览 |
| 试玩 Demo | 否 | 允许匿名体验 |
| 问一个简单问题 | 否 | 允许匿名试用，次数受限 |
| 点击“我也做一个” | 建议注册 | 进入项目创建前弹出轻注册 |
| 保存到“我的项目” | 是 | 项目需要归属账号 |
| 生成成果档案卡 | 是 | 需要绑定项目归属 |
| 复制分享链接 | 是 | 需要稳定链接归属 |

---

## 4. 分享与展示边界

### 4.1 规则

| 层级 | 默认状态 | 触发方式 |
|------|---------|---------|
| 分享链接 | 默认可生成 | 学生主动复制 |
| 灵感墙展示 | 默认关闭 | 学生主动开启 |

### 4.2 展示边界

1. 灵感墙默认不展示真实姓名和头像
2. 默认展示成果档案卡摘要，不直接暴露完整代码仓库
3. 学生可随时关闭灵感墙展示

---

## 5. 轻项目升级为标准研学映射

### 5.1 原则

升级不是重开，已有内容尽量继承，避免重复输入。

### 5.2 映射总览

| 轻项目步骤 | 标准研学映射 |
|-----------|-------------|
| Step 1 想法与方向 | `stage_00` ~ `stage_02` |
| Step 2 设计与实现 | `stage_03` ~ `stage_07` |
| Step 3 展示与反思 | `stage_08` |

### 5.3 MVP 边界

1. 轻项目数据可保留
2. 标准研学可读取已有数据
3. 系统可生成一版初始草稿

## 6. 数据层统一约束：审计字段

```text
AuditFields {
  createdAt: timestamp
  createdBy: string
  updatedAt: timestamp
  updatedBy: string
  deletedAt: timestamp?
  deletedBy: string?
  isDeleted: boolean
}
```

约束说明：

1. 所有时间字段统一采用 `YYYY-MM-DD HH:MM:SS.fff`（UTC）
2. 审计字段由所有核心表统一继承，不在每张表重复展开
3. 业务流程状态字段不混入审计字段

---

## 7. 数据层统一约束：分享与展示字段

```text
ShareFields {
  shareToken: string?
  visibility: "private" | "link" | "wall"
  sharedAt: timestamp?
}
```

约束说明：

1. `shareToken` 用于分享链接
2. `visibility` 用于表达私有/链接可见/灵感墙展示
3. `sharedAt` 记录最近一次开启分享或展示的时间

---

## 8. 一句话结论

本附件只服务学生主闭环，目标是让“起步、完成、分享、升级”四个关键动作可定义、可实现、可验证。
