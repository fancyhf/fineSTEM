## 风险与注意事项

1. **数据库迁移**：删除升学相关 Model 需要数据库迁移脚本，确保生产数据安全
2. **API 兼容性**：删除 API 前确认无其他服务依赖
3. **渐进式重构**：第二阶段涉及大量页面重构，建议按页面逐个对齐，每完成一个页面即验证
4. **ZeroClaw 可用性**：第五阶段依赖 ZeroClaw Gateway 部署就绪，如未就绪可先用 FastAPI 代理层 mock
5. **测试先行**：每个阶段完成后必须运行现有测试 + 新增 E2E 测试验证
