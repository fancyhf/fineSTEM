# Track E: AI Data Analysis Starter Kit (Design Draft)

> **核心理念**: 让 AI 充当“数据保洁员”和“首席分析师”，学生充当“决策者”和“讲故事的人”。

## 1. 项目概览
*   **项目名称**: **“我的第一份数据侦探报告” (My First Data Detective Report)**
*   **难度等级**: Level 1 (入门)
*   **适用年级**: 7-10年级
*   **核心痛点**:
    *   社会实践/研究性学习需要处理问卷数据，但不会用 Excel 公式。
    *   历史/地理课需要分析趋势，但画不出好看的图表。
*   **交付物**: 一个包含互动图表的 Web 页面 (HTML) 或一份 PDF 分析报告。

## 2. 功能模块 (Web Terminal 侧)

### 模块 A: 数据导入区 (The "Raw Material")
*   **支持格式**: Excel (.xlsx), CSV, 纯文本 (粘贴)
*   **AI 预处理 (Magic Cleaner)**:
    *   自动识别表头。
    *   自动检测脏数据（如“年龄”列出现了“未知”）。
    *   *AI 引导语*: "哎呀，第 5 行的数据好像有点奇怪，它是空的，你想让我怎么处理？删掉它还是填个平均数？"

### 模块 B: 对话式分析 (The "Analyst")
*   **交互模式**: 学生提问 -> AI 生成 Python (Pandas/Matplotlib) 代码 -> 实时渲染图表。
*   **预设 Prompt (Starter Questions)**:
    *   "帮我画一个饼图，看看大家都喜欢什么颜色。"
    *   "分析一下，男生和女生谁更喜欢打篮球？"
    *   "帮我找出成绩最好的 3 个班级。"
*   **教学切入点**:
    *   当 AI 生成代码时，高亮显示关键参数（如 `color='red'`, `kind='bar'`）。
    *   *AI 解释*: "看，这行代码里的 `kind='bar'` 就是告诉电脑我们要画柱状图。如果你改成 `line` 会发生什么？"

### 模块 C: 故事生成 (The "Storyteller")
*   **功能**: 基于图表，生成一段“人话”结论。
*   **示例**: "从图上看，虽然 7 班的总分最高，但是 3 班的平均分在过去一个月提升最快，这可能和他们最近的...有关。"

## 3. 示例场景：丝绸之路贸易分析 (Cross-Disciplinary)

### 场景描述
学生拿到一份《马可·波罗游记》的电子文本（或模拟的古代账本数据）。

### 任务清单
1.  **数据清洗**: "把文本里的地名都提取出来，数数哪个地方出现的次数最多。" -> AI 生成词频统计代码。
2.  **可视化**: "把这些地名画在地图上，用圆圈大小表示出现次数。" -> AI 生成 PyEcharts/Folium 代码。
3.  **发现规律**: "为什么这些点都连成了一条线？" -> AI 引导学生思考地理位置与贸易路线的关系。

## 4. 关键技术栈
*   **前端**: React + ECharts / Chart.js
*   **后端**: Python (Pandas + LLM Code Interpreter)
*   **安全**: 数据仅在浏览器端处理或沙箱环境中运行，不存储敏感数据。

## 5. 短视频脚本钩子 (Hook)
*   **标题**: 《历史老师惊呆了！我用 AI 复原了 1000 年前的贸易地图》
*   **画面**:
    *   开头：枯燥的古文书影（黑白）。
    *   中间：快速敲击键盘，AI 代码飞过。
    *   结尾：一张绚丽的动态交互地图亮起，鼠标悬停显示详细数据。
