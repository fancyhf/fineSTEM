## 3. 新发现的严重问题

### N-1: 迁移链不可用（阻塞部署）

| 项 | 详情 |
|----|------|
| 位置 | 000002_add_core_tables.py + 000003_add_counseling_and_skill_tables.py |
| 问题 | 000002 已包含 demos/skill_states/evidence 的完整列定义，000003 又 `op.add_column` 添加相同列。从空库顺序执行 000001→000002→000003 会在 000003 报 "duplicate column" 错误 |
| 影响 | 新部署无法执行迁移，数据库初始化失败 |
| 修复 | 重写迁移链：000002 只创建表和基础列，000003 添加扩展列和新表 |

### N-2: EvidenceModel.author_id ORM 缺失

| 项 | 详情 |
|----|------|
| 位置 | models.py EvidenceModel |
| 问题 | 迁移定义 `author_id NOT NULL`，但 ORM 模型无此字段。repo 用 `created_by` 代替，ORM 插入时 author_id 为 NULL |
| 影响 | 违反 NOT NULL 约束，插入证据时数据库报错 |
| 修复 | 在 EvidenceModel 中添加 `author_id` 字段映射 |

### N-3: 迁移与 ORM 默认值/类型不一致

| 项 | 迁移定义 | ORM 定义 |
|----|---------|---------|
| projects.current_stage 默认值 | `stage_00_bootstrap` | `stage_01_brainstorm` |
| skill_states.light_step 类型 | `Integer` | `String(8)` |

### N-4: PDF 字体路径硬编码 C 盘

| 项 | 详情 |
|----|------|
| 位置 | document_service.py L177-L180 |
| 问题 | 字体搜索路径仅 `C:/Windows/Fonts/`，违反项目规范 |
| 修复 | 增加 D:/H: 备选路径或环境变量配置 |
