## 1. 本版调整原则

本版统一采用“**直接面向学生自助使用**”的产品边界，二期仅保留学生侧能力与核心闭环能力。

### 1.1 范围边界

1. 仅保留学生账号与学生侧功能
2. 不引入多角色账户体系
3. 不引入内容治理工作流
4. 不引入运营控制台类能力

### 1.2 二期核心目标

1. 修复 P0/P1 技术债，确保当前 MVP 主闭环稳定可用
2. 强化学生主路径：`Demo -> 我也做一个 -> 轻项目 -> 成果档案卡 -> 分享链接`
3. 补齐标准研学最小可用能力（数据层优先，UI 渐进）
4. 集成 AI 对话，支持项目上下文引导
5. 实现在线代码编辑与运行（JS 优先）

## 4. 详细执行步骤

### Step 0：技术债修复（P0，必须先完成）

#### 0.1 成果档案卡 Schema 对齐

- 统一对齐到 `V3.3` 字段口径：
  - `title`
  - `oneLiner`
  - `problemSolved`
  - `methodUsed`
  - `screenshots`
  - `reflection`
  - `capabilityTags`
  - `projectMode`
- 前后端类型定义与接口响应统一命名，不再保留双口径字段。

#### 0.2 MemoryDatabase 方法补全与签名修复

- 补齐：
  - `list_projects`
  - `count_projects`
  - `advance_skill_state`
  - `count_evidence`
  - `update_evidence`
- 修正：
  - `update_skill_state` 参数签名
  - `delete_project` 的 `deleted_by` 处理
  - `list_evidence` 增加筛选参数

#### 0.3 查询与安全修复

- `DemoListQuery` 增加 `search` 字段
- `SECRET_KEY` 改为环境变量读取
- 补充 `.env.example` 的安全配置项

#### 0.4 Step 0 验收标准

1. 注册/登录/鉴权可用
2. Demo 列表筛选和搜索可用
3. “我也做一个”可创建项目
4. 轻项目 3 步可保存
5. 成果档案卡可创建并分享
6. 关键 API 不再出现缺失方法异常

### Step 1：学生主闭环强化（P0）

- 聚焦链路：`Demo -> fork -> 轻项目 -> 成果档案卡 -> 分享`
- 完成项：
  - Demo 卡片动作一致化（试玩/拆解/我也做一个/保存项目）
  - fork 模板最小可改版（先覆盖 3 个初始 Demo）
  - 轻项目完成后自动进入成果档案卡生成
  - 分享链接稳定生成与可访问校验

### Step 2：标准研学最小可用（P0/P1）

- P0（数据层）：
  - 9 阶段状态数据结构
  - 阶段保存接口
  - 阶段推进接口
  - 进度查询接口
- P1（UI 层）：
  - 标准研学流程容器
  - 阶段导航与最小表单
  - 渐进补齐 9 阶段页面细节

### Step 3：AI 对话集成（P1）

- 先做 HTTP 聊天接口
- 注入项目上下文（`projectId/currentStage/mode`）
- 对接 ZeroClaw Gateway
- 网关不可用时返回明确错误，不提供“模拟 AI 回复”

### Step 4：在线代码编辑器（P1）

- `Monaco Editor`
- JS 优先（浏览器内执行优先）
- Python 运行后置（避免二期引入复杂服务端沙箱）
- 提供代码保存与运行结果查看

### Step 5：项目下载打包（P2）

- 生成项目包
- 下载至本地 AI IDE
- 补最小 README 与运行说明

### Step 6：测试与质量保障（贯穿执行，里程碑收口）

- 后端：认证、Demo、项目、成果档案卡、聊天核心接口测试
- 前端：关键页面与核心组件测试
- Step 0 与 Step 1 完成后，先跑一次完整回归
