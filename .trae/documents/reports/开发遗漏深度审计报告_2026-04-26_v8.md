## 3. 新发现的问题

### N-1: ExploreDemoDetail 违反 React Hooks 规则

| 项 | 详情 |
|----|------|
| 位置 | ExploreDemoDetail.tsx L146-L154 |
| 问题 | `useEffect` 被放在 `if (loading)` 条件返回块内部，违反 Hooks 规则 |
| 影响 | React 运行时警告和不可预测的行为 |
| 修复 | 将 useEffect 移到组件顶层 |

### N-2: 迁移与 ORM 新不一致

| 项 | 迁移定义 | ORM 定义 |
|----|---------|---------|
| evidence 表缺 `title` 列 | 000002 无此列 | EvidenceModel 有 `title: String(255)` |
| demos 表多 `is_public`/`submitted_at` | 000002 有此两列 | DemoModel 无此两字段 |
| skill_states.version 长度 | String(32) | String(16) |

### N-3: 双重证据创建

| 项 | 详情 |
|----|------|
| 位置 | orchestrator.py L74-75 + assistant_dialogues.py L92-105 |
| 问题 | 同一次对话产生两条证据：orchestrator 写 `text_log`，assistant_dialogues 写 `auto_ai_summary` |
| 影响 | 证据冗余，用户困惑 |

### N-4: AIAssistantDialogues 仍双重发送

| 项 | 详情 |
|----|------|
| 位置 | AIAssistantDialogues.tsx L86-99 |
| 问题 | ws.onclose 中仍调用 REST API 重复发送消息 |
| 影响 | 消息被处理两次 |

### N-5: AIChatPanel 内部信息泄露

| 项 | 详情 |
|----|------|
| 位置 | AIChatPanel.tsx L77-83 |
| 问题 | trace_id、model 名称、工具调用记录暴露给用户 |
| 影响 | 安全和体验问题 |

## 6. 综合评估

### 完成度趋势

| 版本 | 完成度 | 主要进展 |
|------|--------|---------|
| v5 | 85% | 端点级评估（高估） |
| v6 | 60% | 发现 ORM/导出严重问题 |
| v7 | 72% | ORM 补齐 + PDF/DOCX 修复 + WebContainer |
| v8 | **76%** | 轻注册接入 + 灵感墙发布 + 下载格式补齐 |

### 核心瓶颈

1. **AI 流式对话**：伪流式实现影响所有 AI 交互体验，需要 provider 支持 SSE + 后端逐 token 转发
2. **轻项目升级**：30% 完成度，缺少默认映射、数据迁移和前端 UI
3. **迁移一致性**：多处迁移与 ORM 不对齐，新部署可能失败

### 距离 MVP 闭环

MVP 三项最小成功条件：
- ✅ 学生能从 Demo 进入自己的项目（轻注册已接入）
- ⚠️ 学生能完成至少一个轻项目（AI 需配置 API Key）
- ✅ 学生愿意生成并分享成果档案卡（发布到灵感墙已实现）

**MVP 闭环完成度：2.5/3 ≈ 83%**
