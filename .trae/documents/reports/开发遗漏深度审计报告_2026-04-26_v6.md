# fineSTEM V3.3 开发遗漏深度审计报告

- version: v6.0
- created_at: 2026-04-26
- maintainer: AI Agent
- 评估方法: 逐文件代码审查 + 产品规格逐项对比

---

## 1. 总体评估

| 维度 | 完成度 | 说明 |
|------|--------|------|
| 后端 API 端点 | **95%** | 58 个端点全部实现，但多个端点内部实现不完整 |
| 后端数据持久化 | **55%** | ORM 模型严重缺失字段，10 个 Schema 无 ORM/迁移 |
| 前端页面 | **82%** | 15 个核心页面完整，7 个扩展页面仅基础 CRUD |

| 文档导出 | **40%** | MD/JSON/ZIP 可用，PDF 中文乱码，DOCX 无样式 |

**综合可用完成度：约 60%**（端点存在 ≠ 功能可用）

## 2. 严重遗漏清单（阻塞 MVP 闭环）

### P0-1: ZeroClaw 未接入真实 LLM

| 项 | 详情 |
|----|------|
| 位置 | [orchestrator.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/services/orchestrator.py) L96-L137 |
| 现状 | `ZEROCLAW_GATEWAY_URL` 默认 None，`ZEROCLAW_ENABLE_MOCK_FALLBACK` 默认 False。未配置 API Key 时，AI 对话直接抛 RuntimeError |
| 影响 | 所有 AI 对话功能不可用：首页对话、AI Codex、Agent WS、AI 助手对话 |
| 修复 | 配置 `ZEROCLAW_API_KEY` 环境变量，或启用 `ZEROCLAW_ENABLE_MOCK_FALLBACK=True` |

### P0-2: ORM 模型与 Schema 严重不对齐

| 模型 | 缺失字段 | 影响 |
|------|---------|------|
| **DemoModel** | `grade_range`, `tags`, `iframe_url`, `screenshots`, `demo_video_url`, `project_breakdown`, `minimal_replica`, `fork_template_id` | Demo 详情页大量字段为空；fork-template 无法返回真实模板 |
| **ProjectModel** | `from_demo_id`, `initial_data` | Fork 项目无法记录来源 Demo；项目初始数据丢失 |
| **SkillStateModel** | `version`, `light_step`, `stages`, `metadata`, `light_to_standard_mapping` | 状态机细粒度数据无法持久化；轻项目步骤数据丢失 |
| **EvidenceModel** | `author_id`, `content_url`, `related_step` | 证据无法关联创建者；内容链接和关联步骤丢失 |

### P0-3: 10 个 Schema 无 ORM 模型/迁移

| Schema | 文件 | 现状 |
|--------|------|------|
| `SkillRecord` | skills.py | 仅 memory_db，重启丢失 |
| `HongKongMacaoPlan` | counseling.py | 仅 memory_db |
| `InternationalPlan` | counseling.py | 仅 memory_db |
| `ProfileEnhancementPlan` | counseling.py | 仅 memory_db |
| `KnowledgeSource` | counseling.py | 仅 memory_db |
| `QuestionnaireTemplate` | counseling.py | 仅 memory_db |
| `QuestionnaireResponse` | counseling.py | 仅 memory_db |
| `AssistantDialogueSession` | counseling.py | 仅 memory_db |
| `AssistantDialogueMessage` | counseling.py | 仅 memory_db |
| `AuditLogItem` | counseling.py | 仅 memory_db |

**影响**: 这些数据仅存于 JSON 文件，不在 SQL 数据库中，无法被 Alembic 迁移管理，无法在生产环境可靠持久化。

### P0-4: PDF 导出对中文完全不可用

| 项 | 详情 |
|----|------|
| 位置 | [document_service.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/services/document_service.py) L171 |
| 现状 | `raw.encode("ascii", errors="replace")` 将所有中文字符替换为 `?` |
| 影响 | 生成的 PDF 对中文用户毫无价值 |
| 修复 | 引入 `reportlab` + 中文字体 或 `weasyprint` |

### P0-5: DOCX 导出无样式无中文字体

| 项 | 详情 |
|----|------|
| 位置 | [document_service.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/services/document_service.py) L125-L166 |
| 现状 | 手工拼接 OOXML，无样式/字体/字号/页边距设置，标题与正文视觉无区别 |
| 影响 | 导出的 DOCX 排版等同于纯文本 |
| 修复 | 引入 `python-docx` 库 |

## 3. 功能性遗漏（影响体验完整性）

### P1-1: WebSocket 流式对话是伪实现

| 项 | 详情 |
|----|------|
| 位置 | [agent.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/api/agent.py) L98-L107 |
| 现状 | 先同步等待 LLM 完整回复，再用 `build_stream_tokens()` 切分为 20 字符块逐个发送 |
| 影响 | 用户需等 LLM 生成完整回复后才开始看到第一个 token，不是真正的流式体验 |
| 修复 | 从 LLM 逐 token 接收并立即转发 |

### P1-2: WebSocket 无认证

| 项 | 详情 |
|----|------|
| 位置 | [agent.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/api/agent.py) L73 |
| 现状 | `user_id` 直接从客户端消息读取，任何人可伪造身份 |
| 影响 | 安全漏洞，可冒充任何用户 |
| 修复 | WebSocket 握手时验证 JWT token |

### P1-3: 在线代码编辑器不支持 WebContainer

| 项 | 详情 |
|----|------|
| 位置 | [OnlineCodeStudio.tsx](file:///g:/mediaProjects/fineSTEM/apps/frontend/src/pages/OnlineCodeStudio.tsx) |
| 现状 | 仅支持 HTML/CSS/JS iframe 预览，无法运行 Python/Node.js |
| 影响 | 产品规格要求"JS 优先支持 WebContainer"，当前完全缺失 |
| 修复 | 集成 `@webcontainer/api` |

### P1-4: 代码版本记录未实现

| 项 | 详情 |
|----|------|
| 产品规格 | "扩展自动：代码版本记录" |
| 现状 | 无任何代码版本追踪机制 |
| 修复 | 在 EvidenceModel 中增加 `code_snapshot` 类型，或建立独立的 CodeVersion 模型 |

### P1-5: AI 对话摘要自动回写未实现

| 项 | 详情 |
|----|------|
| 产品规格 | "对话结束时自动触发 AI 对话摘要生成，回写业务库作为证据" |
| 现状 | 对话结束后无自动摘要生成和回写逻辑 |
| 修复 | 在 WebSocket `final` 事件后触发摘要生成，调用 evidence API 写入 |

### P1-6: AIAssistantDialogues 双重发送

| 项 | 详情 |
|----|------|
| 位置 | [AIAssistantDialogues.tsx](file:///g:/mediaProjects/fineSTEM/apps/frontend/src/pages/AIAssistantDialogues.tsx) L86-L99 |
| 现状 | WebSocket 收到回复后 `ws.onclose` 又发一次 REST 请求 |
| 影响 | 可能导致重复消息和资源浪费 |
| 修复 | 移除 `onclose` 中的 REST 调用 |

### P1-7: 项目包下载不完整

| 项 | 详情 |
|----|------|
| 位置 | [projects.py](file:///g:/mediaProjects/fineSTEM/apps/backend/app/api/projects.py) L560-L568 |
| 现状 | ZIP 仅含 MD + JSON，不含代码文件 |
| 产品规格 | "项目包下载（.zip）" 应包含完整项目文件 |
| 修复 | 将项目代码、配置文件等一并打包 |
