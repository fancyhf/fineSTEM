# 引导提问库（Question Bank）

## 1) 脑爆阶段（选 3 个，最多 3 个问题）
- 你最想让用户看到的结果是什么？（一句话）
- 如果只能做 1 个功能，你会选哪个？为什么？
- 你不想做什么？（列 2 个）
- 你现在有什么资源：能联网吗？有开发板吗？
- 你希望这是“展示型”（好看/好玩）还是“有用型”（解决问题）？

## 2) 规格化阶段（选 4–6 个）
- 输入是什么？从哪里来？如果拿不到，有替代吗？
- 输出是什么？怎样算成功？给 2 条可验证标准。
- 最可能卡住的地方是什么？怎么绕开？
- 你准备怎么演示给别人看？（1 分钟）

## 3) 反问 quiz（每 2–3 steps 插 1 个，不阻塞）
### 通用
- 如果把这一步删掉，会发生什么？为什么？
- 你觉得这个变量/状态的作用是什么？
- 为什么我们要先做“最小版本”，再加功能？

### Web
- 如果不做状态管理，页面会出现什么问题？
- 为什么要把 UI 组件拆开？

### Kaggle
- 训练集和测试集为什么要分开？
- 交叉验证能避免什么问题？

### 硬件
- 为什么要先做 DeviceCheck？
- 为什么按钮要防抖（或者用状态机）？

### 课程融合
- 这个活动想让学生掌握哪个知识点？
- 如果把评分标准换成“过程分”，会怎样？
