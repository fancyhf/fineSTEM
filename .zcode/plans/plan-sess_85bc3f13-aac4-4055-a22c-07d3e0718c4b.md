# 生成封面图：三种方式

## 需求
把"生成封面"按钮改为提供三种封面来源的选择面板：
1. **项目截图**（默认）— 从该项目已上传的证据截图中选择
2. **AI 生成** — 调用 CogView 自动生成（已实现）
3. **用户上传** — 用户直接上传一张图片作为封面

## 设计

### 第一步：后端 — 截图公开访问 + 三个端点

**改 `apps/backend/main.py`**：挂载 `/uploads` 为静态目录（匿名访问，和 `/media`/`/demos` 一样），让上传的截图和封面图能被灵感墙/首页匿名加载。

**改 `apps/backend/app/api/evidence.py`** 的截图上传端点（`POST /evidence/projects/{pid}/screenshots`，L209）：`content_url` 从 `/api/v1/files/{id}`（带鉴权）改为公开的 `/uploads/{userId}/{projectId}/{fileId}.{ext}` 路径，让截图可匿名访问。

**改 `apps/backend/app/api/achievement_cards.py`**：
- 新增 `GET /projects/{project_id}/screenshots` — 返回该项目所有 evidence 类型为 screenshot 的记录（id/title/url），供前端展示候选缩略图
- 新增 `POST /{card_id}/upload-cover` — 接收 `UploadFile`，存到 `generated/covers/{card_id}.png`（复用 storage_service 的 covers 目录 + `/media` 挂载），写入 `screenshots[0]`，返回更新后的卡片
- 已有的 `POST /{card_id}/generate-cover`（AI 生成）保持不变

### 第二步：前端 — 封面选择面板 UI

**改 `apps/frontend/src/pages/ProjectAchievement.tsx`** 和 **`apps/frontend/src/pages/ProjectDetail.tsx`**：

把"生成封面"按钮改为一个**下拉面板**（参照 Navbar 用户菜单的 `useState + absolute + 透明遮罩关闭` 模式）：

```
┌───────────────────────────┐
│ 选择封面来源               │
│ ┌─────────────────────┐   │
│ │ 📸 项目运行截图（默认）│  │ ← 展开该项目的截图缩略图网格供选择
│ └─────────────────────┘   │
│ ┌─────────────────────┐   │
│ │ ✨ AI 生成封面       │  │ ← 直接调 generateCover
│ └─────────────────────┘   │
│ ┌─────────────────────┐   │
│ │ ⬆ 上传图片          │  │ ← 触发 file input → 调 uploadCover
│ └─────────────────────┘   │
└───────────────────────────┘
```

**三个选项的行为**：
- **项目截图**：调 `listProjectScreenshots(pid)` 拿候选 → 缩略图网格 → 点选 → 调 `setCoverFromScreenshot`。无截图时提示"先去证据面板上传"
- **AI 生成**：调 `generateCover(cardId)`（现有逻辑）
- **上传图片**：隐藏 `<input type="file" accept="image/*">` → 选文件 → 调 `uploadCover(cardId, file)`

**改 `apps/frontend/src/services/api.ts`**：
`achievementCardsApi` 新增：
- `listProjectScreenshots(projectId)` → `GET /achievement-cards/projects/{pid}/screenshots`
- `uploadCover(cardId, file)` → `POST /achievement-cards/{cardId}/upload-cover`（FormData）
- 已有 `generateCover(cardId)` 保持不变

setCoverFromScreenshot 复用已有的 `update`（PATCH）端点传 `screenshots` 字段。

**改 `apps/frontend/src/types/index.ts`**：新增 `ScreenshotOption` 类型（id/title/url）。
